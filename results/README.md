# Results

Test results for the Pi 4B audio workstation. All data below was collected
pre-D-040 on Raspberry Pi 4B running Debian 13 Trixie with stock PREEMPT
kernel, CamillaDSP v3.0.1, and PipeWire. For current architecture (PipeWire
filter-chain convolver) benchmarks, see BM-2 lab notes.

## Directory Structure

```
results/
  benchmarks/    # T1a-T1e: CamillaDSP CPU and thermal measurements (pre-D-040)
  latency/       # T2a-T2b: End-to-end audio latency measurements
```

## benchmarks/

CamillaDSP processing-load and CPU-temperature snapshots for various
chunksize / FIR-filter-length combinations (2-channel stereo, 48 kHz).
Generated by `scripts/test/run_benchmarks.sh`, which queries the CamillaDSP
websocket API and reads `/sys/class/thermal`.

Lab notes: `docs/lab-notes/US-001-camilladsp-benchmarks.md`

### API response files

Each `*_api.txt` file contains the CamillaDSP API state dump captured during the
test run. Key fields: `ProcessingState`, `Processing load` (CPU %), `Buffer level`,
`Clipped samples`.

| File | Test | Configuration | CPU Load |
|------|------|---------------|----------|
| `T1a_api.txt` | T1a | 16k taps, chunksize 2048, 2 ch | 5.23% |
| `T1b_api.txt` | T1b | 16k taps, chunksize 512, 2 ch | 10.42% |
| `T1c_api.txt` | T1c | 16k taps, chunksize 256, 2 ch | 19.25% |
| `T1d_api.txt` | T1d | 8k taps, chunksize 512, 2 ch | 6.35% |
| `T1e_api.txt` | T1e | 32k taps, chunksize 2048, 2 ch | 6.61% |

### Temperature files

Each `*_temp.txt` file contains a single integer: the Pi CPU temperature in
millidegrees Celsius at time of measurement (read from
`/sys/class/thermal/thermal_zone0/temp`). Divide by 1000 for degrees Celsius.

| File | Test | Temperature |
|------|------|-------------|
| `T1a_temp.txt` | T1a | 67679 (67.7 C) |

## latency/

End-to-end audio latency measurements using impulse-detection across a loopback
path. Generated by `scripts/test/measure_latency_v2.py`, orchestrated via
`scripts/test/run_t2a.sh`. Each JSON file contains per-run details and summary
statistics across 5 iterations.

Lab notes: `docs/lab-notes/US-002-latency-measurement.md`

| File | Test | Audio Path | Mean Latency | Best Latency | Std Dev |
|------|------|------------|--------------|--------------|---------|
| `T2a_summary.json` | T2a | PipeWire loopback | 90.63 ms | 85.67 ms | 2.50 ms |
| `T2b_summary.json` | T2b | ALSA direct | 29.24 ms | 20.46 ms | 4.51 ms |

### JSON structure

Each summary file contains:

- `test_id` -- Test identifier (T2a / T2b).
- `config` -- CamillaDSP config file path used for the test.
- `sample_rate` -- Sample rate in Hz (48000).
- `num_runs` -- Number of measurement iterations (5).
- `latencies_ms` -- Array of per-run precise latency values.
- `best_latency_ms` -- Minimum observed latency.
- `mean_latency_ms` -- Arithmetic mean across all runs.
- `std_latency_ms` -- Standard deviation across all runs.
- `cdsp_info` -- CamillaDSP state at time of measurement.
- `runs[]` -- Per-run detail: peak positions, SNR, noise RMS, sample-precise latency.
