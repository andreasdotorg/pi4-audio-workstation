## D-063: 8-Channel Filter-Chain Convolver with Universal Audio Gate (2026-03-30)

**Context:** The previous filter-chain architecture (D-040, D-051) used a
4-channel convolver for the four speaker outputs (L HP, R HP, sub1 LP,
sub2 LP). Headphone and IEM channels (5-8) bypassed the convolver entirely
via direct PipeWire links managed by GraphManager. This created two routing
paths: convolver-routed (speakers) and bypass-linked (monitoring). D-062
introduced FoH passthrough as a built-in speaker profile using Dirac
(identity) coefficients, but left the channel count unresolved.

Three options were considered:

- **Architect recommendation (4ch):** Keep the existing 4ch convolver
  topology. FoH passthrough uses Dirac on 4 speaker channels; headphone
  and IEM channels continue via bypass links.
- **Audio Engineer recommendation (2ch):** FoH passthrough on 2 channels
  only (L+R mains). Sub channels muted in FoH mode because sending
  full-range Dirac to subwoofer amplifiers violates D-031 (mandatory
  subsonic driver protection) — no HPF is present in Dirac coefficients.
- **Owner decision (8ch):** All 8 USBStreamer output channels route through
  the filter-chain convolver. Channels that do not require crossover or
  room correction (headphones, IEM) use Dirac coefficients. A mandatory
  universal audio gate prevents any signal from reaching outputs until
  the operator explicitly opens it.

The AE's D-031 concern about unprotected full-range signal on sub channels
is valid for FoH passthrough mode. The owner's resolution: the universal
audio gate ensures no signal reaches any output without operator
confirmation, making the D-031 risk an operational concern (operator
awareness) rather than a system design gap.

**Decision:** Adopt an 8-channel filter-chain convolver with a mandatory
universal audio gate that applies to ALL profiles.

### 8-Channel Convolver Architecture

All 8 USBStreamer output channels pass through the filter-chain convolver.
Coefficient files per channel:

| Channel | Own-Speaker Profile | FoH Passthrough Profile |
|---------|-------------------|----------------------|
| 1 (L main) | Combined HP + room correction FIR | Dirac (identity) |
| 2 (R main) | Combined HP + room correction FIR | Dirac (identity) |
| 3 (Sub 1) | Combined LP + room correction FIR | Dirac (identity) |
| 4 (Sub 2) | Combined LP + room correction FIR | Dirac (identity) |
| 5 (Eng HP L) | Dirac (identity) | Dirac (identity) |
| 6 (Eng HP R) | Dirac (identity) | Dirac (identity) |
| 7 (Singer IEM L) | Dirac (identity) | Dirac (identity) |
| 8 (Singer IEM R) | Dirac (identity) | Dirac (identity) |

This eliminates bypass links entirely. GraphManager routing simplifies to
a single pattern: app outputs connect to convolver inputs, convolver outputs
connect to USBStreamer. No special-case routing for headphone or IEM channels.

### Universal Audio Gate

ALL profiles (FoH passthrough AND own-speaker) start with `Mult=0.0` on
all gain nodes. No audio flows until the operator explicitly opens the gate.
This is a per-boot safety reset — the `audio_enabled` field in `active.yml`
resets to `false` on every boot. Audio is enabled only after operator
confirmation and completion of the ramp-up sequence.

**Cosine ramp-up mechanism:**

- Duration: 3 seconds
- Steps: 30 (100ms per step via `pw-cli` Mult parameter updates)
- Formula: `gain(t) = 0.5 * (1 - cos(pi * t / T))` where T = ramp duration
- Properties: smooth first and second derivatives (C1 and C2 continuous),
  avoids speaker cone excursion transients from step discontinuities
- Emergency mute: immediate `Mult=0.0` on all gain nodes, available
  throughout ramp-up and normal operation

**Gate opening prerequisites:**
- For FoH passthrough: operator confirmation only (Dirac coefficients,
  no gain staging needed beyond the gate itself)
- For own-speaker profiles: operator confirms gain staging values,
  crossover FIR coefficients loaded, D-031 mandatory HPF validated in
  coefficients, thermal limiter active (if US-092 implemented)

### `active.yml` — `audio_enabled` Field

```yaml
audio_enabled: false  # resets to false on every boot
```

- Written to `true` only after operator confirms AND ramp-up completes
- Per-boot reset ensures the system never auto-plays audio after reboot
- Survives profile switches within a session (gate stays open once opened,
  unless profile switch triggers D-053 safety flow which re-gates)

### GraphManager Simplification

The 8ch convolver eliminates the dual-path routing model:

**Before (4ch + bypass):**
- App → convolver inputs (4 speaker channels)
- App → direct link to USBStreamer (4 monitoring channels)
- GM must manage two routing patterns and coordinate bypass links

**After (8ch):**
- App → convolver inputs (all 8 channels)
- Convolver outputs → USBStreamer (all 8 channels)
- GM routing table is a single uniform pattern
- Mode transitions only change which app (Mixxx/Reaper/signal-gen) links
  to the convolver input ports

### D-031 Implications

In FoH passthrough mode, sub channels (3-4) receive full-range Dirac
(identity) signal — no HPF is baked into the coefficients. This means:

- If the connected subwoofer amplifier is active and the audio gate is
  opened, full-range content reaches the sub drivers
- D-031's mandatory HPF protection is present only in own-speaker profile
  coefficients (where crossover FIR includes the HPF)
- The universal audio gate mitigates this: the operator must explicitly
  confirm before any signal flows, and is expected to understand the
  FoH passthrough implies no driver protection filtering
- Own-speaker profiles remain fully D-031 compliant — crossover FIR
  coefficients always include the mandatory HPF

**Rationale:** (1) Uniform 8ch routing eliminates an entire class of
bypass-link bugs and simplifies GraphManager. (2) The universal audio gate
is strictly safer than the previous D-062 split model (FoH at Mult=1.0,
own-speaker at Mult=0.0) — now ALL profiles start muted. (3) The cosine
ramp-up avoids transients that could damage speakers or amplifiers during
unmute. (4) Per-boot `audio_enabled` reset guarantees no accidental audio
after power cycle. (5) The 8ch architecture is forward-compatible with
any future channel topology changes without routing logic modifications.

**Consequences:**

- D-051 amended: config generator produces 8 convolver nodes for all
  profiles (not topology-dependent count). Dirac coefficients are
  build-time artifacts shipped with the FoH passthrough profile.
- D-053 amended: profile switching safety flow now includes the universal
  audio gate re-arm. Switching from FoH to own-speaker (or vice versa)
  triggers mute → reload → re-gate sequence.
- D-062 amended: FoH passthrough profile uses 8ch Dirac (not 4ch).
  The Mult default in generated `.conf` is `0.0` for ALL profiles
  (previously FoH was `1.0`).
- D-031: FoH passthrough is a documented exception — sub channels receive
  full-range signal but the audio gate prevents uncontrolled output.
  Own-speaker profiles remain fully D-031 compliant.
- US-113 acceptance criteria updated to reflect 8ch + universal gate.
- GraphManager routing table simplified — single routing pattern for
  all profiles and modes.

- "8 channels passthrough from app to output"
- "Mandatory universal audio gate before any sound comes out"
- "ALL profiles start muted, no exceptions"
