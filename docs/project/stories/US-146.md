## US-146: Browser E2E for Venue Select + Gate Open/Close

**As** the QE,
**I want** browser-driven E2E tests for venue selection and audio gate
open/close through the web UI,
**so that** the venue/gate workflow (US-113) is tested end-to-end through the
actual user interface.

**Status:** draft
**Priority:** P1
**Effort:** ~1 worker day
**Origin:** QE E2E audit (session 14)
**Related:** US-113 (FOH passthrough profile, audio gate)

### Acceptance Criteria

- [ ] Browser test opens the venue/config page
- [ ] Selects a venue via browser UI interaction
- [ ] Opens the audio gate via browser UI interaction

- [ ] Closes the audio gate via browser UI interaction
- [ ] Verifies gate closed state reflected in UI
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
