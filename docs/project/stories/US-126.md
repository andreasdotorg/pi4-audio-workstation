## US-126: Persistent Audio Gate Banner

**As** the system operator,
**I want** a prominent always-visible banner when the audio gate is closed,
**so that** I always know when no audio is reaching the speakers, regardless of
which UI tab I'm viewing.

**Status:** draft
**Priority:** High — AD flagged gate visibility as critical safety UX gap
**Effort:** 0.5 worker days
**Origin:** Owner decision (session 9) + AD review (gate indicator only in Config tab is insufficient)
**Depends on:** US-113 (audio gate infrastructure), US-123 (boot state — gate always closed on boot)
**Blocks:** none
**Decisions:** D-063 (universal audio gate)

### Background

Currently the audio gate indicator is only visible in the Config tab. The AD flagged
this as a critical gap: an operator on the Dashboard, Spectrum, or Graph tab has no
indication that the gate is closed and no audio is reaching the speakers. This is
especially important during boot (gate always closed per US-123), after emergency
mute (watchdog), and during crash recovery.

The banner must be visible on ALL tabs — it is the operator's primary awareness
indicator for audio output state.

### Acceptance Criteria

- [ ] **AC #1: Full-width red banner on ALL tabs when gate is closed.** Banner
      appears at the top of every tab (Dashboard, Spectrum, Graph, Config, Test,
      Measurement, System). Visually prominent: red background, white text, full
      viewport width.

- [ ] **AC #2: Banner text and action.** Text: "AUDIO GATE CLOSED — No audio
      output". Includes a link or button that navigates to Config > Venue gate
      controls. Single tap to reach the gate open action from any tab.

- [ ] **AC #3: Banner disappears when gate opens.** When the operator opens the
      gate (cosine ramp-up completes), the banner is removed without page refresh.
      Uses WebSocket push from GM gate state change event.

- [ ] **AC #4: Banner reappears on gate close.** Any event that closes the gate
      triggers the banner: operator manual close, emergency mute (watchdog timeout),
      reboot (US-123 gate-closed invariant), GM mode transition to Standby. Banner
      appears immediately via WebSocket push — no polling.

- [ ] **AC #5: Covers all scenarios.** Banner is correct for: first boot (no venue,
      gate closed), subsequent boot (venue persisted, gate closed pending restore),
      mid-gig crash recovery (gate closed, venue restorable), emergency mute
      (watchdog closed gate), operator manual mute.

- [ ] **AC #6: Does not obstruct critical UI.** Banner is fixed-position at top,
      pushing page content down (not overlaying). On small screens, banner text
      truncates gracefully. Banner does not interfere with status bar or mode
      dropdown.
