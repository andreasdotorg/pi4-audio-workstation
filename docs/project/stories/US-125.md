## US-125: Explicit Mode Arming

**As** the system operator,
**I want** mode selection and audio gate opening to be separate deliberate actions,
**so that** I can configure the routing topology and verify setup before any audio
reaches the speakers.

**Status:** draft
**Priority:** High — safety-critical separation of routing and output
**Effort:** 1 worker day
**Origin:** Owner decision (session 9) + architect/AD advisory input
**Depends on:** US-123 (deterministic boot state), US-113 (audio gate infrastructure)
**Blocks:** none
**Decisions:** D-063 (universal audio gate), D-011 (dual quantum)

### Background

Mode selection (DJ, Live, Standby) changes the link topology, quantum, and may launch
applications (Mixxx, Reaper). But mode selection must NOT open the audio gate — the
operator must explicitly open the gate as a separate action after verifying the routing
is correct. This two-step "arm then fire" pattern prevents accidental audio output
during mode transitions.

Per architect input: no race condition between mode selection and gate opening — GM
processes RPCs single-threaded, so mode change completes fully before any gate-open
RPC can execute.

### Acceptance Criteria

- [ ] **AC #1: Mode selection triggers routing + quantum + app launch.** Status bar
      mode dropdown (DJ / Live / Standby) sends GM `set_mode` RPC. GM reconfigures
      link topology, sets quantum (DJ=1024, Live=256, Standby=256), and launches the
      associated application if applicable.

- [ ] **AC #2: Mode selection does NOT open audio gate.** After mode change, all
      gain nodes remain at their current state. If gate was closed (Mult=0.0), it
      stays closed. Mode selection never calls `open_gate`.

- [ ] **AC #3: Gate opening is a separate explicit action.** The "Open Gate" control
      lives in Config > Venue section (not in the mode dropdown). Opening the gate
      applies venue gain values with cosine ramp-up (D-063).

- [ ] **AC #4: Gate button disabled when no venue loaded.** If no venue config is
      active (`venue_loaded: false` in `get_state`), the "Open Gate" button is
      disabled with tooltip: "Load a venue configuration first." This prevents
      opening the gate with undefined gain staging.

- [ ] **AC #5: Quantum set automatically by mode selection.** DJ mode sets quantum
      1024 (via `pw-metadata`). Live mode and Standby set quantum 256. No separate
      quantum control needed — it follows the mode.
