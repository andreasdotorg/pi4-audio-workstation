## US-122: Unified Real-Time Measurement Display

**As** the audio system operator tuning a venue,
**I want** a single web UI view showing transfer function, coherence, and delay for all speaker channels simultaneously,
**so that** I can see the complete system measurement state at a glance without switching between views or running separate per-channel measurements.

**Status:** draft
**Priority:** Medium — usability layer over US-120 and US-121 data
**Effort:** 2-3 worker days
**Origin:** Owner-approved architect analysis (session 9)
**Depends on:** US-120 (real-time transfer function with coherence), US-121 (simultaneous multi-channel delay)
**Blocks:** none
**Decisions:** D-040 (PW filter-chain), D-001 (combined minimum-phase FIR)
**Theory:** `docs/theory/realtime-transfer-function.md` (section 4.1, Phase 1 item 5),
`docs/theory/multichannel-delay-measurement.md` (section 5, display)

### Background

US-120 provides per-channel transfer function and coherence data. US-121 provides
per-channel propagation delays. This story combines them into a unified measurement
display — the operator places the mic, hits "Measure All," and immediately sees
transfer function + coherence + delay for all 4 speaker channels in one view.

This replaces the current workflow where each measurement aspect (frequency response,
delay, verification) is a separate step with separate tools. The unified display is
the operator's primary interface during the "place mic, measure, adjust, verify"
venue setup phase.

### Acceptance Criteria

- [ ] **AC #1: Multi-channel transfer function overlay.** Up to 4 transfer function
      magnitude traces overlaid on a single plot (20Hz-20kHz, log frequency axis,
      dB magnitude axis with auto-ranging and manual override). Channels color-coded
      and labeled: L main (blue), R main (red), Sub 1 (green), Sub 2 (orange).
      Traces individually toggleable.

- [ ] **AC #2: Per-channel coherence display.** Coherence (0-1) shown as trace opacity
      or background shading beneath the magnitude plot, matching channel colors.
      High coherence (> 0.85) = solid trace, moderate (0.5-0.85) = semi-transparent,
      low (< 0.5) = faded/dotted. Frequencies with low coherence visually de-emphasized
      in the magnitude plot to communicate "do not trust this data."

- [ ] **AC #3: Delay sidebar.** Per-channel delay values in a sidebar panel: channel
      name, measured delay (ms, 1 decimal), delay in samples, relative delay
      (referenced to furthest speaker = 0). Updates in real time (at least 1 Hz).
      Color-coded status: green (good coherence + stable delay), yellow (moderate
      coherence or delay drift), red (low coherence or no signal). "Capture snapshot"
      button freezes current state.

- [ ] **AC #4: Phase display (optional toggle).** Phase (degrees) vs frequency on a
      secondary plot or Y-axis. Hidden by default (magnitude + coherence is primary).
      Phase only meaningful where coherence > 0.5 — grayed out elsewhere.

- [ ] **AC #5: "Measure All" one-button workflow.** Single button: GM enters
      measurement mode, signal-gen starts multi-channel uncorrelated noise (US-121),
      transfer function engine starts computing (US-120 + US-121), display populates
      within 2-5 seconds. "Stop" button halts noise, GM returns to previous mode.
      Last measurement snapshot remains visible until explicitly cleared.

- [ ] **AC #6: Target curve and correction preview.** Target response curve overlay
      (flat by default, or user-defined house curve from file). Deviation trace:
      difference between measured TF and target, per channel. Visual indicator of
      which frequency bands would be corrected (where Cxy > threshold and deviation >
      correction threshold from D-009). Optional: overlay current FIR filter magnitude
      for "what the system is doing" vs "what the room looks like."

- [ ] **AC #7: Snapshot and export.** "Capture" button freezes current measurement
      state as a named snapshot with timestamp. Snapshots can be overlaid on live
      measurement for A/B comparison (e.g., before vs after speaker repositioning).
      Snapshots persisted to venue data directory. Export measurement data as JSON
      (transfer functions, coherence, delays) for archival or offline analysis.

- [ ] **AC #8: Responsive layout.** Display works on 1920x1080 desktop (VNC) and
      degrades gracefully on tablet-sized screens (1024x768). Critical information
      (magnitude traces, delay values) always visible without scrolling. Collapsible
      sidebar, full-screen measurement mode that hides other UI elements.

### Risks

1. **Visual clutter.** 4 overlaid magnitude traces + coherence + target curve can become
   unreadable. Mitigation: individual trace toggles, coherence shown as opacity not
   separate traces, phase hidden by default. UX review critical.
2. **Canvas rendering performance.** Updating 4 traces at 10 Hz with 4096 frequency bins
   each = ~160K data points/sec. Mitigation: Canvas2D or WebGL rendering (not SVG),
   line decimation (only render visible pixels, not all bins), requestAnimationFrame
   with skip logic.
3. **Data synchronization.** Transfer function data (US-120) and delay data (US-121)
   arrive on different update cadences. Mitigation: display uses latest available data
   per panel, with staleness indicators tied to graph clock timestamps (US-077).
4. **VNC latency.** wayvnc over WiFi adds 50-200ms display latency. "Real-time" display
   is relative — operator sees updates with VNC delay. Not a problem for tuning workflow
   (human reaction time >> VNC latency).
