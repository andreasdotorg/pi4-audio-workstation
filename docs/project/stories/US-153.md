## US-153: Browser E2E for D-009 Gain Ceiling Runtime Enforcement (Safety)

**As** the QE,
**I want** a browser-driven E2E test that verifies the D-009 gain ceiling is
enforced at runtime -- attempting to set gain above the ceiling is rejected,
**so that** the safety-critical gain limit cannot be bypassed through the UI.

**Status:** draft
**Priority:** P1 (safety-critical)
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Related:** D-009 (gain ceiling), gain_integrity.rs

### Acceptance Criteria

- [ ] Browser test opens the config/gain page
- [ ] Attempts to set gain above the D-009 ceiling via UI interaction
- [ ] Verifies the gain is clamped or rejected (not applied above ceiling)
- [ ] Verifies gain integrity check remains passing
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
