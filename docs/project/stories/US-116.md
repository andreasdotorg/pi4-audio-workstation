### Background

Multi-speaker systems (mains + subs) have drivers at different distances from the listening position. Without delay compensation, signals arrive at different times, causing comb filtering in the crossover region. The solution: delay closer speakers so all arrivals are synchronized. The furthest speaker is the reference (zero delay); all others get positive delay.

Existing infrastructure:
- `room_correction/time_align.py` — arrival detection + delay computation (17 tests passing)
- `room_correction/pw_config_generator.py` — generates PW filter-chain `delay` builtin nodes
- `runner.py` — pipeline integration, saves `delays.yml`

Missing: venue.yml integration, GM runtime delay control, AE-recommended detection improvements.

### AE-Recommended Detection Improvements (consulted session 7)

1. **Threshold: -6 dB (not -20 dB).** -20dB is only 5dB above noise floor in marginal measurements. -6dB is the half-energy point — physically meaningful, 34dB above noise.

2. **Backward-from-peak search.** Current implementation searches forward from sample 0, vulnerable to pre-onset noise spikes. Search backward from the IR peak to find the -6dB onset crossing — guarantees finding the direct sound onset.

3. **Hybrid detection: threshold for mains, group delay at crossover for subs.** Sub IRs have dispersed onsets (slow LF wavefront, ported enclosure group delay). Threshold detection is ambiguous. Group delay evaluated at the crossover frequency (from speaker profile YAML: `crossover.frequency`, currently 80Hz) directly measures the delay causing comb filtering.

4. **Inter-channel measurement gaps: 1s for mains, 3s for subs.** Room modes at sub frequencies (RT60 = 1-3s) contaminate subsequent measurements without adequate decay time.

### Delay Compensation Architecture

- **Where:** PW filter-chain `delay` builtin node, per-channel. Signal chain: `conv -> gain -> delay`. Only added for channels with delay_ms > 0.
- **NOT baked into FIR coefficients.** Separate concern, runtime-adjustable, negligible CPU (circular buffer vs FFT).
- **Runtime control:** `pw-cli s <convolver-id> Props '{ params = [ "delay_<suffix>:Delay" <value> ] }'` — same mechanism as gain Mult.
- **Measurement procedure:** Sequential per-channel sweep using signal-gen channel bitmask (F-097). Same sweep that produces room correction IRs also provides arrival time data.
