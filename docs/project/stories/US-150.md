## US-150: Browser E2E for Gate Banner Cross-Tab Visibility

**As** the QE,
**I want** a browser-driven E2E test that verifies the audio gate banner is
visible across all tabs when the gate is closed,
**so that** the safety-critical gate status is always visible to the operator
regardless of which tab they are viewing.

**Status:** draft
**Priority:** P1
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Related:** US-113 (audio gate), D-063 (universal audio gate)

### Acceptance Criteria

- [ ] Browser test opens the web UI with gate closed (default state)
- [ ] Verifies gate banner is visible on the dashboard tab
- [ ] Navigates to config tab -- gate banner still visible
- [ ] Navigates to other tabs -- gate banner still visible
- [ ] Opens gate -- banner disappears on all tabs
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
