## US-144: Browser E2E for Panic Button Click

**As** the QE,
**I want** a browser-driven E2E test that clicks the panic button in the status
bar and verifies VU meters go silent, then recovers,
**so that** the safety-critical panic mute is tested end-to-end through the
actual user interface.

**Status:** draft
**Priority:** P1
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Related defects:** F-282 (test_mute_e2e.py partially mislabeled)

### Acceptance Criteria

- [ ] Browser test opens the dashboard or control page
- [ ] Clicks the mute button via browser interaction
- [ ] Verifies VU meters go to zero/silent after mute
- [ ] Clicks unmute
- [ ] Verifies VU meters recover to non-zero values
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
