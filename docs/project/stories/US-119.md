## US-119: Tier 3 — Build-Time Overrides for Mesa and PipeWire

**As** the system builder,
**I want** to build Mesa without LLVM/llvmpipe and PipeWire without Bluetooth
and libcamera support,
**so that** the SD card image closure shrinks by ~500-800 MiB by excluding
libraries for hardware and features that are explicitly unused on the Pi 4B
audio workstation.

**Status:** draft
**Priority:** Low (owner-directed, image size reduction — do after Tier 1+2)
**Effort:** 2-3 worker days (custom builds + validation)
**Origin:** Bloat analysis of US-072 SD card image — Mesa LLVM and PipeWire
transitive dependencies identified as significant closure contributors
**Depends on:** US-072 (NixOS build), US-117 (Tier 1 — do trivial wins first)
**Related:** US-114 (minimal kernel config), US-118 (Tier 2 — Reaper closure)
**Decisions:** D-019 (Bluetooth disabled), D-022 (hardware V3D GL)

### Background

The NixOS SD card image includes Mesa with full LLVM/llvmpipe support and
PipeWire with Bluetooth and libcamera support. None of these are needed:

- **Mesa LLVM**: The Pi 4B uses the V3D hardware GPU driver (D-022). The
  llvmpipe software renderer is explicitly unused — Mixxx runs with hardware
  GL at ~85% CPU vs 142-166% with llvmpipe. LLVM alone contributes ~522 MiB
  to the closure.
- **PipeWire Bluetooth**: Bluetooth audio is disabled by design (D-019).
  The Hercules DJ controller uses USB-MIDI, not Bluetooth audio. The bluez
  dependency in PipeWire adds ~100-200 MiB to the closure.
- **PipeWire libcamera**: No cameras are connected to the audio workstation.
  libcamera and its dependencies add ~100-200 MiB.

### Changes

**1. Mesa without LLVM (~522 MiB savings)**

Override the Mesa package with `enableLlvmpipeDriver = false` (or equivalent
nixpkgs override argument). This removes LLVM from the Mesa closure while
retaining the V3D and VC4 hardware drivers.

```nix
# Example (actual override syntax depends on nixpkgs version)
nixpkgs.overlays = [(final: prev: {
  mesa = prev.mesa.override {
    enableLlvmpipeDriver = false;
  };
})];
```

**2. PipeWire without Bluetooth/libcamera (~200-400 MiB savings)**

Override the PipeWire package to disable unused optional dependencies:

```nix
nixpkgs.overlays = [(final: prev: {
  pipewire = prev.pipewire.override {
    bluezSupport = false;    # D-019: BT disabled
    libcameraSupport = false; # No cameras
  };
})];
```

The exact override argument names must be verified against the current
nixpkgs PipeWire derivation. PipeWire's `default.nix` in nixpkgs exposes
these as function arguments.

### Acceptance Criteria

- [ ] Mesa override applied: LLVM/llvmpipe not in image closure
- [ ] V3D hardware GL works on test Pi (Mixxx launches, wlinfo shows V3D)
- [ ] labwc compositor runs with hardware V3D (no pixman fallback)
- [ ] PipeWire override applied: bluez not in image closure
- [ ] PipeWire override applied: libcamera not in image closure
- [ ] PipeWire audio pipeline fully functional: filter-chain convolver loads,
      all 8 channels route correctly, GraphManager mode transitions work
- [ ] WirePlumber starts without errors (no bluez plugin warnings)
- [ ] Image closure size reduced by >= 400 MiB compared to pre-US-119
- [ ] `nix build .#images.sd-card` succeeds (cross-compilation from x86_64)
- [ ] No regressions in existing US-072 acceptance criteria
- [ ] Overrides documented: rationale, how to revert if needed, nixpkgs
      version compatibility notes

### Risks

1. **Mesa V3D breakage** — removing LLVM might cause Mesa's build system to
   also disable or break the V3D driver. The V3D driver does not use LLVM
   (it has its own compiler backend), but the nixpkgs Mesa derivation may
   have unexpected coupling. Mitigation: test thoroughly on test Pi; verify
   with `wlinfo` and Mixxx before production.
2. **Build time increase** — custom Mesa and PipeWire builds are not in the
   NixOS binary cache (cache.nixos.org). They must be built from source,
   adding significant time to the image build. Mesa alone can take 30+
   minutes. Mitigation: cache builds locally or on a binary cache.
3. **Nixpkgs update friction** — overrides may break when nixpkgs is updated
   (argument names change, dependencies restructured). Mitigation: pin
   nixpkgs in flake.lock; review overrides on each nixpkgs bump.
4. **WirePlumber bluez module** — WirePlumber may expect bluez libraries at
   runtime even if PipeWire was built without them. Mitigation: test
   WirePlumber startup logs for errors; may need a corresponding WirePlumber
   override.
5. **PipeWire override argument names** — the exact override arguments for
   bluez and libcamera must be verified against the current nixpkgs
   derivation. They may differ from the examples above. Mitigation: read
   `pkgs/servers/pipewire/default.nix` in nixpkgs before implementing.
