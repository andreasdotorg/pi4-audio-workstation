## US-147: WebSocket Smoke Test -- All 5 WS Endpoints Connect and Stream in Browser

**As** the QE,
**I want** a browser-driven smoke test that verifies all 5 WebSocket endpoints
connect and stream data through the web UI,
**so that** WebSocket connectivity regressions are caught early.

**Status:** draft
**Priority:** P1
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)

### Acceptance Criteria

- [ ] Browser test opens the web UI
- [ ] Connects to all 5 WebSocket endpoints via the browser:
  - `/ws/pcm` (PCM spectrum data)
  - `/ws/monitoring` (VU level data)
  - `/ws/status` (system status)
  - `/ws/siggen` (signal generator state)
  - 5th WS endpoint (TBD -- verify current endpoint list)
- [ ] Verifies each WebSocket receives at least one message within timeout
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)

### Structural Recommendation

The QE E2E audit recommends moving all mislabeled tests (F-278 through F-283)
to a new `tests/service-integration/` directory to physically separate
service-integration tests from genuine browser-driven E2E tests. This prevents
future misclassification and makes coverage metrics accurate.
