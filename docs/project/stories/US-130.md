## US-130: Expose Test Runners as Flake Packages

**As** the CI pipeline,
**I want** test runners exposed as `packages` outputs in `flake.nix` (in addition
to the existing `apps` outputs),
**so that** `nix build` can pre-build and cache test closures before execution,
enabling explicit closure materialization and parallel builds in CI.

**Status:** BACKLOG
**Depends on:** US-070 (CI pipeline)
**Blocks:** none
**Related:** T1 CI fix (removed broken `nix build --no-link` on app outputs)

### Problem

The flake exposes test runners as `apps` only (e.g., `.#test-all`,
`.#test-pcm-bridge`). These are `writeShellScript` wrappers that reference
store paths via string interpolation. `nix build` on an app output builds
the wrapper script but may not reliably materialize the full closure
(the referenced store paths are string interpolations, not direct build
inputs of the wrapper derivation).

This prevents a useful CI optimization: pre-building all test closures
in parallel via `nix build --no-link` before running them sequentially.
Currently, each `nix run` builds its closure on demand, sequentially.
While Nix store deduplication means later suites reuse deps from earlier
ones, the first suite pays the full build cost, and there is no way to
warm the cache for all suites in parallel.

### Solution

Add `packages.<system>.test-*` outputs that expose the underlying test
environments (Python environments with test deps, Rust build artifacts,
Playwright environment) as buildable derivations. The existing `apps`
outputs continue to work for `nix run` — the packages are an additional
output for `nix build` use cases.

### Acceptance Criteria

- [ ] Each test runner has a corresponding `packages` output in `flake.nix`
- [ ] `nix build .#test-all-env` (or similar naming) materializes the full
  closure needed by `nix run .#test-all`
- [ ] `nix build --no-link .#test-all-env .#test-pcm-bridge-env ...` succeeds
  and Nix parallelizes the builds
- [ ] CI can optionally use a pre-build step: `nix build --no-link` then
  `nix run` (the `nix run` becomes near-instant since everything is cached)
- [ ] Existing `apps` outputs unchanged — `nix run .#test-all` continues to work

### Definition of Done

- [ ] Package outputs added to `flake.nix`
- [ ] CI workflow updated with optional pre-build step (or documented as
  available for future use)
- [ ] Verified: `nix build` on package outputs materializes full closures
- [ ] No regression in `nix run` behavior

### Risks

1. **Naming convention:** Need to avoid collision between app and package
   names. Consider `test-all-env` or `test-all-closure` for packages.
2. **Maintenance burden:** Two outputs per test suite (app + package) that
   must stay in sync. Mitigate by having the app reference the package.
