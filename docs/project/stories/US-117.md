## US-117: Tier 1 — Trivial Image Size Reductions

**As** the system builder,
**I want** to remove unnecessary firmware, locales, packages, and flake
registry bloat from the NixOS SD card image,
**so that** the image closure shrinks by ~1.1 GiB with zero functional impact
on the audio workstation.

**Status:** draft
**Priority:** High (owner-directed, image size reduction)
**Effort:** 0.5 worker days
**Origin:** Bloat analysis of US-072 SD card image (6.59 GiB uncompressed,
4.9 GiB closure)
**Depends on:** US-072 (NixOS build — must have working baseline)
**Related:** US-114 (minimal kernel config — same goal, different layer)

### Background

The US-072 SD card image is 6.59 GiB uncompressed (1.95 GiB zstd). Bloat
analysis identified four changes that are trivial, safe, and have no
functional impact on a dedicated Pi 4B audio workstation.

### Changes

**1. Firmware trim (~730 MiB savings)**

`hardware.enableRedistributableFirmware = true` (`nix/nixos/hardware.nix:25`)
installs the full linux-firmware collection (736 MiB). The Pi 4B only needs
Broadcom WiFi/BT firmware (`brcm/` + `cypress/`). Replace with explicit
firmware package list targeting only Pi-required firmware (~7 MiB).

**2. Locale trim (~217 MiB savings)**

Full glibc locale archive is ~222 MiB. The workstation only uses
`en_US.UTF-8`. Add `i18n.supportedLocales = [ "en_US.UTF-8/UTF-8" ]` to
reduce to ~5 MiB.

**3. Remove git from systemPackages (~54 MiB self, ~364 MiB closure savings)**

`git` is listed in `nix/nixos/configuration.nix:109`. A dedicated audio
workstation does not need git at runtime. If maintenance requires git, it can
be installed temporarily via `nix-env -iA nixpkgs.git` or `nix shell nixpkgs#git`.

**4. Kernel source / flake registry removal (~186 MiB savings)**

The flake registry references the kernel source (186 MiB) in the closure.
Fix via `nix.registry` override or `nix.channel.enable = false` to prevent
the kernel source from being pulled into the image closure.

### Acceptance Criteria

- [ ] `hardware.enableRedistributableFirmware` replaced with Pi-only firmware
      packages in `nix/nixos/hardware.nix`
- [ ] WiFi works on the test Pi after firmware trim (Broadcom 43455 adapter)
- [ ] `i18n.supportedLocales` set to `[ "en_US.UTF-8/UTF-8" ]` in
      `nix/nixos/configuration.nix`
- [ ] `git` removed from `environment.systemPackages` in
      `nix/nixos/configuration.nix`
- [ ] Flake registry / channel configuration does not pull kernel source into
      the image closure
- [ ] Image closure size < 3.9 GiB (down from 4.9 GiB — ~1 GiB reduction)
- [ ] `nix build .#images.sd-card` succeeds
- [ ] All audio services start correctly (PipeWire, GraphManager, pcm-bridge,
      signal-gen, web UI)
- [ ] No regressions in existing US-072 acceptance criteria

### Risks

1. **WiFi firmware missing** — if the Pi-only firmware list is incomplete,
   WiFi may not work at venues. Mitigation: test on test Pi before production.
   Broadcom 43455 is the only WiFi chip; required firmware files are well
   documented.
2. **Locale edge case** — if any application (Mixxx, Reaper) requires a
   locale other than en_US.UTF-8, it may fail. Mitigation: both applications
   are tested with en_US.UTF-8 on the current Debian setup.
3. **Git needed for maintenance** — removing git from the image means
   `nix-env` or `nix shell` is needed for any git operations on the Pi.
   Mitigation: this is a dedicated appliance, not a development machine.
