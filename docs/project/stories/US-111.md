## US-111: Local-Demo PipeWire Graph Topology Redesign

**As** a developer testing the web UI and audio pipeline locally,
**I want** the local-demo to use a room-simulator filter-chain that replaces
both the USBStreamer output and UMIK-1 capture with identical node names,
a `spa-node-factory` clock driver, and loopback-based ada8200-in capture,
**so that** the local-demo faithfully exercises the production PipeWire graph
topology without WirePlumber, with realistic per-channel room acoustics,
and with reliable process cleanup.

**Status:** ready (architect design complete, AE signed off, 2026-03-29)
**Priority:** Medium (development workflow quality)
**Effort:** 1-2 worker days
**Origin:** Architect redesign proposal (session 3)
**Depends on:** US-075 (local PW test env — COMPLETE)

### Background

The current local-demo (`nix run .#local-demo`) uses WirePlumber with policy
disabled plus null-audio-sink adapter nodes for virtual hardware. This has
several problems:

1. WirePlumber is needed solely for adapter node port activation — it adds
   process complexity and is a source of zombie processes (F-216)
2. Null-audio-sink nodes don't behave identically to production hardware
   nodes — different port naming, different format negotiation
3. No acoustic simulation — the local-demo passes audio through Dirac
   (identity) impulses, so the spectrum display shows the raw signal rather
   than what would appear in a real room
4. UMIK-1 capture path untested — no simulated microphone input

The architect has produced a complete redesign that eliminates these issues.

### Design Summary (Architect)

**Core idea:** A single room-simulator filter-chain replaces both the
USBStreamer output sink AND the UMIK-1 capture source, using identical
PipeWire node names so GM's routing table works unmodified.

**Key components:**

1. **Room-sim filter-chain** — a PipeWire filter-chain module that:
   - Has a capture (sink) side with `node.name` matching USBStreamer output
     prefix (`alsa_output.usb-MiniDSP_USBStreamer...`)
   - Has a playback (source) side with `node.name` matching UMIK-1 capture
     prefix
   - Internally convolves each input channel with a per-channel room impulse
     response (4 IRs: left speaker, right speaker, sub1, sub2 — each with
     different propagation delays and room characteristics)
   - Sums the convolved outputs to produce a simulated mono microphone signal
   - The simulated mic signal appears on the source side, exercising the
     full measurement capture path

2. **Clock driver via `spa-node-factory`** — a `support.node.driver` node
   that provides the graph clock. This replaces WirePlumber's role in clock
   management. The driver node is self-activating and requires no session
   manager.

3. **ada8200-in via loopback module** — a PipeWire loopback module creates
   the ada8200-in capture node. Loopback modules are self-activating (no WP
   needed for port activation).

4. **WirePlumber eliminated** — with `spa-node-factory` for the clock driver
   and loopback for ada8200-in, WirePlumber is no longer needed in
   local-demo. This simplifies the process tree and addresses F-216 (zombie
   accumulation).

### Audio Engineer Requirements (signed off)

The AE approved the design with 6 requirements:

1. Per-channel room IRs must model different speaker positions (different
   propagation delays, different early reflection patterns)
2. Sub channels must have appropriate LF room gain (~3-6 dB boost below
   80 Hz from room modes)
3. Propagation delays must be physically plausible (1-5ms range for typical
   venue distances)
4. The simulated microphone signal must be mono (single-channel sum of all
   speaker contributions)
5. Room IRs should be short enough for low CPU overhead (512-1024 taps at
   48 kHz = 10-21ms, sufficient for early reflections)
6. The simulation need not be physically accurate — it must be "good enough"
   to exercise the measurement pipeline and produce visually meaningful
   spectrum/level displays

### Technical Notes

- Room IRs can be generated with a simple Python script using scipy: exponential
  decay envelope, random early reflections, per-channel delay offset, LF shelf
  for sub channels
- The `spa-node-factory` approach for clock driver is documented in PipeWire
  wiki — used by headless/embedded deployments that don't need WirePlumber
- Filter-chain config format is the same as the production convolver config —
  just different coefficient files and node names
- This design is a stepping stone toward US-067 (full physics-based simulation)
  but much simpler — no T/S parameter modeling, no ray tracing, just
  representative IRs
