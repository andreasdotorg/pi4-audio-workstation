## US-149: Browser E2E for Mode Switch via Dropdown

**As** the QE,
**I want** a browser-driven E2E test that switches modes via the browser
dropdown and verifies the mode transition completes,
**so that** mode switching is tested through the actual user interface.

**Status:** draft
**Priority:** P1
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Depends on:** local-demo stack, US-140 (reconciler settlement for deterministic waits)

### Acceptance Criteria

- [ ] Browser test opens the web UI
- [ ] Selects a different mode from the mode dropdown
- [ ] Verifies mode transition completes (mode badge updates)
- [ ] Verifies no error state after transition
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
