## US-152: Browser E2E for Watchdog Latch/Unlatch Cycle (Safety)

**As** the QE,
**I want** a browser-driven E2E test that exercises the watchdog latch and
unlatch cycle through the web UI,
**so that** the safety-critical watchdog mechanism is verified end-to-end.

**Status:** draft
**Priority:** P1 (safety-critical)
**Effort:** ~1 worker day
**Origin:** QE E2E audit (session 14)
**Related:** watchdog.rs, gain_integrity.rs

### Acceptance Criteria

- [ ] Browser test triggers a watchdog latch condition (e.g., gain integrity
      violation or simulated fault)
- [ ] Verifies watchdog latch state reflected in the UI (warning/error
      indicator)
- [ ] Verifies audio is muted while watchdog is latched
- [ ] Unlatches the watchdog via the UI
- [ ] Verifies recovery -- watchdog unlatched, audio can resume
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
