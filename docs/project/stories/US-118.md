## US-118: Tier 2 — Reaper Closure Optimization

**As** the system builder,
**I want** to reduce or eliminate the ~800 MiB of unnecessary closure pulled
in by the Reaper package's transitive dependencies,
**so that** the SD card image is significantly smaller without losing Reaper
functionality for live vocal performance mode.

**Status:** draft
**Priority:** Medium (owner-directed, image size reduction)
**Effort:** 0.25 worker days (one-line override + build verification)
**Origin:** Bloat analysis of US-072 SD card image — Reaper's transitive
dependency on VLC identified as single largest source of closure bloat
**Depends on:** US-072 (NixOS build)
**Related:** US-117 (Tier 1 — trivial reductions, do first)

### Background

The nixpkgs `reaper` package pulls in VLC (1.4 GiB closure) via its
`LD_LIBRARY_PATH` wrapper. The dependency chain: Reaper → VLC → Samba →
Qt5 → ffmpeg 7 → GTK4 → gst-plugins-bad.

**Root cause (investigated):** Reaper is a commercial binary. The nixpkgs
package wraps it with `wrapProgram` adding VLC to `LD_LIBRARY_PATH`.
Reaper's `reaper_video.so` plugin `dlopen`'s `libvlc.so.5` at runtime for
**video decoding only**. All audio functionality (recording, mixing,
JACK/PipeWire, MIDI, plugin hosting) is completely independent of VLC.
Reaper handles missing VLC gracefully — it shows "VLC install path not
found" in video decoder preferences and operates normally.

The VLC closure contributes **142 store paths / ~857 MiB** exclusive to
it (not shared with Mixxx or other system components). This is the single
largest source of image bloat.

### Recommended approach: VLC stub override

Replace VLC with an empty stub package. The nixpkgs `reaper` derivation
takes `vlc` as a function argument, so `.override` works directly:

```nix
# In nix/nixos/applications.nix, replace:
reaper
# With:
(reaper.override {
  vlc = pkgs.runCommand "vlc-stub" {} "mkdir -p $out/lib";
})
```

This gives VLC's `dlopen` an empty lib directory — the load fails gracefully,
Reaper disables its video decoder path, and all audio functionality works
unchanged. One line, no patches, no maintenance burden beyond tracking
nixpkgs `reaper` argument changes.

### Acceptance Criteria

- [ ] Reaper overridden with VLC stub in `nix/nixos/applications.nix`
- [ ] VLC and its 142 unique transitive dependencies not in the image closure
- [ ] Reaper launches correctly: `pw-jack reaper` starts, audio I/O works
- [ ] Reaper core functions verified: audio recording, playback, MIDI,
      JACK transport
- [ ] Image closure size reduced by >= 500 MiB compared to pre-US-118
- [ ] `nix build .#images.sd-card` succeeds
- [ ] Override documented with rationale in code comment

### Risks

1. **Nixpkgs Reaper argument change** — if a future nixpkgs update renames
   or removes the `vlc` argument, the override breaks. Mitigation: pinned
   flake.lock; CI build check catches this immediately.
2. **Video import disabled** — Reaper's video decoder (`reaper_video.so`)
   will not work. This project does not use video in Reaper (live vocal
   performance with backing tracks only). Mitigation: if video is ever
   needed, remove the override.
3. **Unexpected VLC usage** — some Reaper feature beyond video might use
   VLC. Investigation found zero references to VLC outside `reaper_video.so`.
   Mitigation: test core live-performance workflow on Pi.
