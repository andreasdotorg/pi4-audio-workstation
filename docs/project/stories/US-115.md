## US-115: 8-Channel Filter-Chain Convolver (D-063)

**As** the audio system operator,
**I want** all 8 USBStreamer output channels to route through the PipeWire filter-chain convolver with per-channel gain control and a universal audio gate,
**so that** the routing model is uniform (no bypass links), all channels have safety-gated gain control, and the system is ready for venue-based configuration management.

### Background

The current convolver handles 4 speaker channels (L main, R main, sub1, sub2). Headphone and IEM channels (5-8) bypass the convolver entirely via direct PipeWire links from app to USBStreamer. D-063 mandates all 8 channels route through the convolver:
- Channels 5-8 use Dirac (identity) coefficients — no DSP processing, just passthrough
- ALL 8 gain nodes start at Mult=0.0 (universal audio gate per D-063)
- Bypass links are eliminated entirely — GM uses a single uniform routing pattern

- [ ] **AC #4: HP/IEM through convolver.** DJ mode: Mixxx HP channels -> convolver AUX4-5 -> USBStreamer ch 5-6. Live mode: Reaper HP channels -> convolver AUX4-5, IEM channels -> convolver AUX6-7 -> USBStreamer ch 5-8.

- [ ] **AC #6: All existing tests pass.** All gain_integrity tests updated for 8 gains. All routing tests updated (bypass links removed, new convolver links added). All watchdog tests cover 8 channels. Full test suite passes: `nix run .#test-all`.
- [ ] **AC #7: Local-demo functional.** `nix run .#local-demo` starts with 8ch convolver. GM creates correct 8ch routing. Level-bridge-hw-out taps all 8 convolver output channels. Web UI dashboard shows all 8 channels in meters.
