## US-142: Browser E2E for Spectrum Display

**As** the QE,
**I want** a browser-driven E2E test that verifies the spectrum display renders
live data from the PCM WebSocket relay,
**so that** regressions like F-275 (spectrum display gone after US-131 merge) are
caught by the test suite before shipping.

**Status:** draft
**Priority:** P0 (F-275 regression guard)
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Depends on:** F-275 fix (PCM bridge connection), local-demo stack
**Related defects:** F-275 (regression this test would have caught)

### Acceptance Criteria

- [ ] Browser test opens the dashboard page
- [ ] Verifies `/ws/pcm` WebSocket connects through the web UI relay
- [ ] Verifies the spectrum canvas renders non-zero data (not blank/silent)
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
