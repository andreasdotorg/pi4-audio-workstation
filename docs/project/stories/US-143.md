## US-143: Browser E2E for VU Meters

**As** the QE,
**I want** a browser-driven E2E test that verifies VU meters display live level
data from the monitoring WebSocket,
**so that** level metering regressions are caught before shipping.

**Status:** draft
**Priority:** P0
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Depends on:** local-demo stack with level-bridge running

### Acceptance Criteria

- [ ] Browser test opens the dashboard page
- [ ] Verifies `/ws/monitoring` WebSocket delivers level data
- [ ] Verifies VU meter DOM elements show non-zero values
- [ ] Verifies status bar mini meters show non-zero values
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
