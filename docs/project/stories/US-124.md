## US-124: First-Boot Experience

**As** an operator setting up a new workstation (or after state file loss),
**I want** clear indication that no venue is configured and guidance on how to
set one up,
**so that** I understand the system state and can proceed without confusion.

**Status:** draft
**Priority:** Medium — UX polish for new/reset installations
**Effort:** 0.5 worker days
**Origin:** Owner decision (session 9) + UX specialist input
**Depends on:** US-123 (deterministic boot state — `get_state` RPC with `persisted_venue`)
**Blocks:** none
**Decisions:** D-062 (first-boot / active config)

### Background

When no venue has ever been configured (fresh install) or the persisted venue file is
missing/corrupt, the system boots to Standby with no venue loaded. The operator needs
to know this is expected and how to proceed. Per UX input: no wizard needed — a
non-blocking banner pointing to the venue configuration is sufficient.

### Acceptance Criteria

- [ ] **AC #1: "No venue configured" banner.** When `get_state` returns
      `persisted_venue: null`, the Dashboard shows a non-blocking banner:
      "No venue configured — set up your speakers in Config > Venue."

- [ ] **AC #2: Banner links to Config > Venue.** Banner includes a clickable link
      or button that navigates directly to the Config tab's Venue section.

- [ ] **AC #3: Banner is dismissible.** Operator can close the banner for the
      current session. It reappears on next page load / browser refresh if still
      no venue is configured.

- [ ] **AC #4: System fully functional without venue.** Meters, system health,
      spectrum display, and all non-audio-output features work in Standby without
      a venue loaded. The banner is informational, not blocking.

- [ ] **AC #5: Banner disappears on venue load.** Once a venue is loaded (via
      "Restore" button from US-123 or manual venue selection in Config), the
      banner is removed without page refresh.
