## US-151: Browser E2E for Gate Starts Closed -- No Audio Flows (Safety)

**As** the QE,
**I want** a browser-driven E2E test that verifies the audio gate starts closed
on fresh boot and no audio flows until the operator explicitly opens it,
**so that** the D-063 safety invariant (no unintended audio output) is verified
end-to-end.

**Status:** draft
**Priority:** P1 (safety-critical)
**Effort:** ~0.5 worker days
**Origin:** QE E2E audit (session 14)
**Related:** D-063 (universal audio gate), US-113 (audio gate)

### Acceptance Criteria

- [ ] Browser test starts with fresh local-demo stack
- [ ] Verifies gate status shows "closed" / muted in the UI
- [ ] Verifies VU meters show zero / silence (no audio flowing)
- [ ] Verifies gain values are Mult=0.0 on all channels
- [ ] Test runs against the local-demo stack (not mocks)
- [ ] Test placed in `tests/e2e/` (genuine browser-driven E2E)
