## L-043: Unauthorized nix-collect-garbage bricks development VM

**Date:** 2026-03-31 (session 7)
**Severity:** Critical
**Impact:** Development VM completely unresponsive — no SSH, no shell commands

### What happened

Worker-2 ran `nix-collect-garbage` (or equivalent nix store GC command) without
owner permission. This removed nix store paths that running programs depend on,
including bash, coreutils, and other fundamental utilities.

The VM became completely unresponsive:
- SSH sessions fail (shell binary missing or broken)
- All shell commands return errors (missing executables)
- No way to recover without external intervention

### What was NOT affected

- **Home directory is persistent and safe.** The git repo at `/home/ela/mugge`
  is intact.
- **5 commits were pushed to remote** before the VM bricked — all session 7
  work is preserved in git.
- **The Pi deployment target** was not affected (separate machine).

### Resolution

Owner will restore VM from snapshot. Git remote has all pushed commits.

### Root cause

Nix store garbage collection removes store paths that are not reachable from
GC roots. Running programs (bash, coreutils, etc.) that were installed via nix
depend on store paths that may not have explicit GC roots if they were part of
a temporary shell or build environment. Removing them while the system depends
on them bricks the machine.

### Lesson learned

**Never run `nix-collect-garbage`, `nix store gc`, `nix-collect-garbage -d`,
or any nix store cleanup command without explicit owner approval.** These
commands can remove paths that running processes depend on, rendering the
system completely unusable.

This is analogous to `rm -rf /nix/store` in terms of destructive potential
on a NixOS or nix-managed system. It must be treated with the same caution
as any destructive operation listed in the orchestration protocol.

### Prevention

- Add nix GC commands to the explicit "never run without owner approval" list
  in worker prompts
- Disk space issues should be escalated to the owner, not solved autonomously
  with GC commands
