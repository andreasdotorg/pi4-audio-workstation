### Description

`persist_and_load_venue_name` fails with assertion:

```
left == right
left: None
right: Some("test-venue-1")
```

The test persists a venue name and then loads it back, expecting
`Some("test-venue-1")`. Instead it gets `None`, indicating the venue
name was not persisted or not loaded correctly.

Likely pre-existing — not related to US-128 (PipeWire upgrade).

### Root Cause (suspected)

Two possibilities:

1. **Test bug:** The test may not be writing to the correct path, or
   the persist and load operations use different file paths / state
   directories. A temp directory mismatch or missing setup step could
   cause the load to read from a location where nothing was written.

2. **Code bug:** The `persist_venue_name` or `load_venue_name` function
   in `venue.rs` may have a logic error — e.g., writing to a path that
   doesn't exist, failing silently on write errors, or reading from a
   different config key than what was written.

Needs investigation to determine whether this is a test infrastructure
issue or a product bug.

### Fix

1. Investigate `venue.rs` around line 632 — check the persist/load
   paths and any temp directory setup in the test.
2. If test bug: fix the test setup (ensure temp dir exists, paths match).
3. If code bug: fix the persist/load logic in venue.rs.
