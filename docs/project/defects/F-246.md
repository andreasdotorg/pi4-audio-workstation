## F-246: Mixxx invisible in graph viz — classifyNode() drops JACK clients with empty media_class (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Medium (affects production Pi and local-demo)
**Status:** Open
**Affects:** US-095 graph visualization, production Pi graph tab
**Found by:** Architect review (session 7)

### Description

`classifyNode()` in `graph.js` uses the PipeWire `media_class` property
to classify and display nodes in the graph visualization. JACK clients
created via `pw-jack` (including Mixxx) have an empty `media_class`
property. As a result, `classifyNode()` does not recognize them and they
are dropped from the graph — Mixxx's source node is invisible.

This affects both local-demo (Mixxx substitute) and the production Pi
(real Mixxx via `pw-jack`).

### Impact

- Operator cannot see Mixxx in the graph topology during DJ mode
- Graph viz shows an incomplete picture of the audio routing
- Debugging audio path issues is harder when the primary source is invisible
- Affects production, not just local-demo (architect confirmed)

### Root Cause

`pw-jack` wraps JACK clients as PipeWire nodes but does not set the
`media_class` property that native PipeWire nodes have. The graph
renderer's classification logic relies exclusively on `media_class`
and has no fallback for nodes where it is empty or absent.

### Fix

Add a name-based classification fallback in `classifyNode()` for known
applications when `media_class` is empty. For example:
- Node name containing "Mixxx" → classify as audio source
- Node name containing "Reaper" → classify as audio source
- Other unclassified nodes → show with a generic "unknown" category
  rather than dropping them

This preserves the `media_class`-first classification for native PipeWire
nodes while catching JACK clients by name.
