## F-239: Default profile 2way-80hz-ported fails — missing sub-ported-15.yml identity (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Medium
**Status:** Open
**Affects:** US-090 FIR filter generation, local-demo demo data
**Found by:** QE exploratory Playwright session (session 7)

### Description

Selecting the default speaker profile "2way-80hz-ported" and attempting
filter generation fails with: "Speaker identity file not found:
sub-ported-15.yml". The demo data set is incomplete for this profile.

The bose-home profile works correctly — this is a demo data coverage gap
for the default profile only.

### Impact

Operator confusion — the default profile fails on first use. The happy
path works with bose-home, but the default selection gives an error.

### Fix

Add the missing `sub-ported-15.yml` speaker identity file to the demo
data set, or change the default profile to one that has complete data
(e.g., bose-home).
