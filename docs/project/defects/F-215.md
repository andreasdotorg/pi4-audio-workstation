## F-215: numpy empty-array crash in local-demo measurement (2026-03-29)

**Severity:** Low
**Status:** Open
**Found in:** Measurement pipeline (room correction / signal processing)
**Affects:** US-096 (UMIK-1 calibration pipeline), US-097 (Room compensation workflow)
**Found by:** QE exploratory testing (Phase 2, measurement tab in local-demo)

**Description:**

When attempting to use the measurement workflow in local-demo mode (no real UMIK-1
microphone connected), numpy raises a RuntimeWarning or crashes with an error related
to computing statistics (e.g. `np.mean`) on an empty array. This occurs because the
local-demo simulated microphone input produces no valid capture data, resulting in
an empty numpy array being passed to a statistical function.

**Expected:** In local-demo mode without a real microphone, the measurement pipeline
should either gracefully handle the absence of real capture data (returning a
meaningful error or using simulated data) or clearly indicate that measurement
requires a real microphone.

**Actual:** numpy operation on empty array raises a warning/exception, causing the
measurement session to fail with an unhelpful error.

**Root cause:** The measurement pipeline's signal processing code does not guard
against empty capture buffers. In local-demo mode, the simulated audio source does
not produce data in the format/path expected by the measurement capture pipeline,
resulting in zero-length arrays reaching numpy statistical functions.

**Note:** This is expected behavior in the sense that local-demo cannot perform real
measurements. The defect is about error handling — the failure mode should be clean
and informative rather than an unhandled numpy exception.

**Fix:** Add a guard in the measurement signal processing path to check for
empty/zero-length arrays before computing statistics, and return a clear error
message (e.g. "No microphone data captured — check that UMIK-1 is connected").

**Verified:** Not yet.
