## F-217: E2E conftest console error filter misses /ws/pcm 403 in mock mode (OPEN)

**Filed:** 2026-03-30 (session 4)
**Severity:** Medium (test infrastructure — not production code)
**Status:** Open
**Affects:** `src/web-ui/tests/integration/conftest.py:291`
**Found by:** QE (Gate 1 review of US-089 through US-098, session 4)

### Description

The E2E test conftest's console error filter at line 291 only suppresses
`/ws/siggen` 403 errors (expected in mock mode when `PI4AUDIO_SIGGEN` is
not set). However, `/ws/pcm/capture-usb` also returns 403 in mock mode
(no real PCM bridge), and this is not filtered.

This causes:
- 7 E2E test failures (assertion on unexpected console errors)
- 12 teardown errors (same assertion in the `page` fixture teardown)

### Root Cause

`conftest.py:291`:
```python
real_errors = [e for e in console_errors if "/ws/siggen" not in e]
```

This only filters `/ws/siggen`. The `/ws/pcm` endpoints also produce 403
console errors in mock/test mode, but are not included in the filter.

### Expected

All known-benign 403 errors from WebSocket endpoints that are unavailable
in mock mode should be filtered. At minimum: `/ws/siggen` and `/ws/pcm`.

### Fix

Extend the filter at `conftest.py:291` to also suppress `/ws/pcm` 403
errors in mock mode. For example:

```python
benign_patterns = ["/ws/siggen", "/ws/pcm"]
real_errors = [
    e for e in console_errors
    if not any(p in e for p in benign_patterns)
]
```

### Impact

- Blocks clean Gate 1 pass for US-089 (speaker config management web UI)
- 7 false-positive test failures + 12 teardown errors in E2E suite
- No production impact — test infrastructure only
