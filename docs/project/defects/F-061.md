## F-061: `pw-dump` subprocess hangs under WebSocket load ‚Äî event loop saturation (VERIFIED)

**Severity:** High (blocks real-mode Pi operation ‚Äî all PW introspection endpoints hang)
**Status:** VERIFIED (2026-03-29, session 4) ‚Äî all subprocess calls migrated to `asyncio.to_thread(subprocess.run)`
**Found in:** S-005 deploy session (Pi, 2026-03-22)
**Affects:** All backend endpoints using `pw-dump` / `pw-cli` subprocess calls
**Found by:** Worker (S-005 deployment verification)

**Description:** When the web UI has active WebSocket connections (dashboard
polling, status bar updates), `pw-dump` subprocess calls hang indefinitely.
The root cause is event loop saturation: `asyncio.create_subprocess_exec`
runs the subprocess within uvicorn's async event loop, and under WebSocket
load the event loop cannot service the subprocess PIPE reader fast enough.
The subprocess output buffer fills, the subprocess blocks on write, and the
async reader never gets scheduled ‚Äî classic deadlock.

This blocks real-mode verification of F-056 (quantum display) and F-057
(gain controls) on Pi, since both depend on `pw-dump` / `pw-cli` subprocess
calls that hang under normal UI operation.

**Root cause:** Using `asyncio.create_subprocess_exec` with PIPE for
PipeWire CLI tools in an event loop that is saturated by WebSocket handlers.

**Fix:** All 5 PipeWire subprocess call sites replaced
`asyncio.create_subprocess_exec` with `asyncio.to_thread(subprocess.run, ...)`,
running subprocesses in the thread pool executor:

1. `pw_helpers.py:74` ‚Äî `_pw_dump_sync()`: `subprocess.run(["pw-dump"], ...)`
2. `pw_helpers.py:191` ‚Äî `_read_mult_sync()`: `subprocess.run(["pw-cli", "enum-params", ...])`
3. `pw_helpers.py:251` ‚Äî `_set_mult_sync()`: `subprocess.run(["pw-cli", "s", ...])`
4. `config_routes.py:234` ‚Äî `_set_quantum_sync()`: `subprocess.run(["pw-metadata", ...])`
5. `measurement/session.py:1291` ‚Äî `_reload_convolver()`: `subprocess.run(["pw-cli", "destroy", ...])` (called from `_deploy_sync` in `asyncio.to_thread` context at line 1196)

**Verification (session 4, 2026-03-29):** Zero instances of
`asyncio.create_subprocess_exec` remain in `src/web-ui/`. All PW subprocess
calls (`pw_helpers.py`, `config_routes.py`, `measurement/session.py`) use
synchronous `subprocess.run` wrapped in `asyncio.to_thread`. 827 unit tests
pass.
