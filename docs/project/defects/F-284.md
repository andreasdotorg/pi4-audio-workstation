## F-284: test_measurement_workflow.py mislabeled as E2E -- PI_AUDIO_MOCK=1 scope too large

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** RESOLVED
**Resolved by:** PR #30 (commit `893c92c8`)
**Affects:** `src/web-ui/tests/e2e/test_measurement_workflow.py`
**Origin:** QE E2E audit (session 14)

### Description

`test_measurement_workflow.py` is located in the `e2e/` directory but runs
under `PI_AUDIO_MOCK=1`, meaning the mock boundary encompasses too much of
the stack. The tests exercise mocked responses rather than the real local-demo
stack. This is not genuine E2E testing.

### Fix

Reclassify: move to `tests/service-integration/`. If measurement workflow
E2E coverage is needed, write new tests that run against the real local-demo
stack without `PI_AUDIO_MOCK=1`.

### Resolution

Five mislabeled test files (including `test_measurement_workflow.py`) were
moved from `tests/e2e/` to `tests/service-integration/` in the F-284 test
reclassification commit. Shared fixtures were extracted into
`tests/fixtures/backend.py`. Test runners updated accordingly.
