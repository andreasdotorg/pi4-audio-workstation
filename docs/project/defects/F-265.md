# F-265: UMIK-1 sim shows no signal in test tab during measurement mode

**Status:** FIX IN PROGRESS (rework per owner feedback)
**Severity:** Major (blocks manual testing workflow)
**Found:** Session 12
**Component:** web-ui test_tool

## Symptom

When using the test tab to play test tones in measurement mode, the
UMIK-1 spectrum analyzer and SPL readout show silence (flat line / -inf dB).
The signal generator is playing but no signal reaches the simulated UMIK-1.

## Root Cause

The test tab's `ensure-measurement-mode` endpoint (F-144) switches GM to
measurement mode but does NOT open the D-063 audio gate.  The convolver
starts with all Mult=0.0 (D-063 universal audio gate).  Without loading
a venue and opening the gate, signal-gen output is multiplied by zero at
the convolver, producing silence through the entire downstream chain:

    signal-gen -> convolver-in -> [Mult=0.0] -> convolver-out (silence)
      -> room-sim -> UMIK-1 sim (silence)

The venue selection tab and the automated measurement session both have
their own gate-opening logic, but the test tab was missing it entirely.

## Previous Fix (rejected by owner)

The first fix auto-loaded `venues[0]` when no venue was set.  Owner
rejected this: auto-loading a venue is dangerous (ordering is arbitrary,
could load wrong gains).  If a venue was previously active and gate was
closed (e.g., by standby transition), re-opening with the *same* venue
is correct.  If no venue was ever selected, the user must be told to
select one.

## Current Fix (rework)

Modified `_ensure_gate_open()` in `src/web-ui/app/test_tool/routes.py`
with owner-specified logic:

1. If venue set AND gate open -> return `already_open` (no-op)
2. If venue set AND gate closed -> re-open gate with same venue
3. If no venue -> return error telling user to select a venue first

Never auto-loads a venue.  The user must explicitly select one via the
venue tab or API before using the test tab.

## Files Changed

- `src/web-ui/app/test_tool/routes.py` — rewrote `_ensure_gate_open()`
  to check `gate.get("venue")` instead of `has_pending_gains`; removed
  auto-load logic; added `no_venue_selected` error response
