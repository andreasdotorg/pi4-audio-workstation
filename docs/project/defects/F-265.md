## F-265: UMIK sim shows no signal in measurement mode (test tab)

**Filed:** 2026-04-04
**Severity:** Medium
**Status:** OPEN
**Affects:** Measurement mode — web UI test tab
**Related:** F-262 (CI E2E silence — may share root cause), US-085 (GM lifecycle)
**Origin:** Owner report (no signal visible in test tab when measurement mode active)

### Description

When measurement mode is active, the simulated UMIK microphone does not
produce signal visible in the web UI test tab. The level meters show silence.
The owner reported this as a user-facing issue — the test tab should show
signal from the UMIK sim when measurement mode is engaged.

This was originally conflated with F-262 (CI E2E silence). F-262 covers the
CI infrastructure issue (GM reconciler not establishing link topology in the
local-demo stack). F-265 covers the runtime/UI symptom: measurement mode
signal flow does not produce audible output from the simulated UMIK.

### Root Cause (suspected)

The measurement mode signal path requires GM to link the UMIK sim source
through the measurement pipeline to the level-bridge metering ports. If GM
does not create these links (reconciler timing, missing node registration,
or config mismatch), the UMIK sim output goes nowhere and the test tab
shows silence.

Possible sub-causes:
1. **GM measurement mode links not created:** The reconciler may not have
   a measurement mode topology defined, or it may fail to establish it.
2. **UMIK sim not producing signal:** The signal-gen UMIK simulation may
   not be active or routing correctly in measurement mode.
3. **Level-bridge not connected:** The metering bridge may not be linked
   to the measurement path output ports.

### Verification

Owner sees signal in the web UI test tab when measurement mode is active
with the UMIK sim running.

### Impact

Measurement workflow is non-functional — the operator cannot verify signal
flow before starting a measurement session. Blocks room correction pipeline
development.
