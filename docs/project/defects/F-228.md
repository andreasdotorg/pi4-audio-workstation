## F-228: Default gm_mode hardcoded to "dj" in fallback paths — should be "standby" (OPEN)

**Filed:** 2026-03-30 (session 6)
**Severity:** Low
**Status:** Open
**Affects:** Web UI fallback mode display, GM state reporting
**Found by:** QE (session 6, exploratory testing of local-demo)

### Description

Several fallback paths in the web UI and backend hardcode `gm_mode = "dj"`
as the default when the GraphManager state is unavailable or the mode field
is missing. After the session 6 rename from "monitoring" to "standby"
(commit `62105d8`), these should default to `"standby"` — the idle/ready
state — not `"dj"`.

### Impact

Minor display inaccuracy. The system may briefly show "DJ" mode in the UI
when GM hasn't reported its state yet, instead of the correct "Standby"
default.
