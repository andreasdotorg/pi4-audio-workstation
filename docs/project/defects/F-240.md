## F-240: Unknown filter_type "fullrange" not handled by backend (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Medium
**Status:** Open
**Affects:** US-090 FIR filter generation, speaker profile topology handling
**Found by:** QE exploratory Playwright session (session 7)

### Description

Selecting the "workshop-xrk10f-fullrange" speaker profile and attempting
filter generation fails with: "Unknown filter_type 'fullrange'". The
backend crossover/filter generation code does not handle the "fullrange"
topology type.

### Context

Current backend handles 2-way and 3-way topologies. A "fullrange" speaker
(single driver, no crossover) is a valid and simple case — it needs only
room correction, no crossover filtering. The backend should recognize
"fullrange" and generate a correction-only FIR (or passthrough if no
measurement data).

### Fix

Add "fullrange" as a recognized filter_type in the filter generation
backend. For fullrange speakers: skip crossover filter generation, apply
room correction only (or passthrough if no measurement session selected).
