## F-214: Missing identity file for ported profile placeholder (2026-03-29)

**Severity:** Low
**Status:** Open
**Found in:** Speaker identity/profile data (`speakers/identities/` directory)
**Affects:** US-089 (Speaker config management), profile validation
**Found by:** QE exploratory testing (Phase 2, speaker config tab)

**Description:**

When exploring the speaker configuration tab in local-demo, a profile references a
speaker identity file that does not exist in the identities directory. The profile
appears to be a placeholder or ported entry from an earlier configuration format. When
deep validation runs (e.g. during profile activation or the validate endpoint), it
reports the identity as missing.

**Expected:** All profiles shipped with the local-demo or seeded during setup should
reference identity files that exist, or placeholder profiles should use placeholder
identities that are also present.

**Actual:** Profile validation returns an error for the missing identity reference.
The profile is visible in the UI but cannot be activated.

**Root cause:** A profile YAML file references an identity name that was either not
created, not ported from the old config format, or was accidentally omitted from the
test/demo data set.

**Fix:** Either create the missing identity file as a placeholder with reasonable
defaults, or update the profile to reference an existing identity. Low priority since
this only affects demo/test data, not production profiles which are created by the
owner.

**Verified:** Not yet.
