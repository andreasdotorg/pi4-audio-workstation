## F-222: Zombie process accumulation in container dev environment (OPEN)

**Filed:** 2026-03-30 (session 5)
**Severity:** High (1392 zombies observed, blocks owner workflow)
**Status:** Open
**Affects:** All background processes: local-demo.sh, E2E tests (Playwright/Chrome),
nix builds, git, PipeWire, WirePlumber, GraphManager, signal-gen, pcm-bridge, level-bridge
**Found by:** Owner (session 5)

### Description

Background processes launched by Claude Code agents accumulate as zombies when
their parent process is killed (e.g., agent context compaction, task timeout,
Ctrl+C). Observed 1392 zombies in a single session:

|---------|-------|

| chrome_crashpad | 262 |

| wireplumber | 105 |

### Root Cause

The dev environment runs in a container where PID 1 is `sleep infinity` — not
a proper init system. When an agent's shell (the intermediate parent) is killed,
its child processes become orphans reparented to PID 1. Since `sleep infinity`
never calls `wait()`, these orphans become permanent zombies that cannot be
reaped.

Zombies under PID 1 cannot be killed (`kill -9` has no effect on zombies —
they are already dead). `kill -SIGCHLD 1` has no effect because `sleep infinity`
has no SIGCHLD handler. The only resolution is restarting the container.

### Contributing Factors

1. **local-demo.sh** spawns ~8 background processes (PipeWire, WirePlumber,
   GraphManager, signal-gen, level-bridge, pcm-bridge, web-ui, room-sim IR
   generator). When killed without Ctrl+C (which triggers the trap handler),
   children become orphan zombies.

2. **E2E tests** (Playwright) spawn Chrome/Chromium processes. Test timeouts
   or agent kills leave Chrome zombies.

3. **Nix builds** on remote builder spawn SSH + nix processes that zombie on
   agent kill.

### Fix Options

1. **Use a proper container init** (e.g., `tini`, `dumb-init`, or
   `pid1` wrapper) as PID 1 instead of `sleep infinity`. This is the
   correct fix — a proper init reaps orphaned zombies automatically.

2. **Process group cleanup in local-demo.sh** — use `setsid` or process
   groups so that killing the parent also kills all children (prevents
   orphaning). The existing trap handler does this for Ctrl+C but not for
   SIGKILL/SIGTERM from agent termination.

3. **Periodic zombie reaper** — a cron job or background process that
   periodically calls `wait()` on orphaned children. Workaround only.

### Recommendation

Option 1 (proper container init) is the correct fix. Options 2-3 are
defense-in-depth improvements for local-demo.sh specifically.
