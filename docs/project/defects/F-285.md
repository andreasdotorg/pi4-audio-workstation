## F-285: measurement session E2E -- browser START does not transition from IDLE

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** RESOLVED
**Resolved by:** PR #30 (commit `7f72606c` -- F-275 port propagation fix)
**Affects:** `tests/e2e/test_measurement.py::test_full_session_browser`
**Origin:** QE E2E audit (session 14)

### Description

The browser E2E measurement test (`test_full_session_browser`) was xfailed
because clicking START did not transition the session out of IDLE state.
The root cause was that signal-gen was not reachable in the local-demo stack
-- the dynamically allocated signal-gen port was not propagated to the web-ui
process via environment variables.

### Resolution

PR #30's port propagation fix (F-275, commit `7f72606c`) exports
`PI4AUDIO_SIGGEN_HOST` and `PI4AUDIO_SIGGEN_PORT` in the local-demo launcher.
With this fix, the measurement session starts successfully -- signal-gen is
reachable and the session transitions out of IDLE through gain_cal, measuring,
and filter_gen states.

The session now reaches the filter verification stage but ends in ERROR due
to F-262 (room-sim IRs fail min-phase filter verification). The xfail marker
has been updated to reference F-262 as the remaining blocker. F-285 itself
(signal-gen connectivity) is fully resolved.

### Related

- F-275 (port propagation fix -- same PR)
- F-262 (room-sim filter verification -- remaining blocker for measurement completion)
