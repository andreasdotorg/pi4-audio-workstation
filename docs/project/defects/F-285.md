## F-285: Measurement session E2E returns 503 — signal-gen not reachable in local-demo

**Filed:** 2026-04-06
**Severity:** Medium
**Status:** OPEN
**Affects:** E2E tests `test_full_session_api`, `test_full_session_browser` in
`src/web-ui/tests/e2e/test_measurement.py`
**Related:** F-262 (previously masked by F-262 xfails)
**Origin:** T2 E2E test failure during D-262 branch verification

### Description

When running `nix run .#test-e2e`, the measurement session E2E tests fail:

1. `test_full_session_api` — HTTP 503 on `POST /api/v1/measurement/start`.
   The 503 comes from `measurement/routes.py` (`siggen_unavailable`) — the
   signal generator is not reachable when the measurement session attempts
   to start.

2. `test_full_session_browser` — Playwright clicks START, but the measurement
   state badge never leaves IDLE. Timeout after 10s. This is a consequence
   of the same startup failure — the session cannot start, so the state
   never transitions.

The failure is consistent (reproduced in 3 consecutive runs).

### Root Cause (suspected)

The signal-gen service is running in the local-demo stack (verified by other
tests that pass), but the measurement session's startup sequence cannot reach
it. Possible sub-causes:

1. **Signal-gen port not propagated:** The local-demo stack uses dynamic
   ports. The measurement session may be trying to reach signal-gen on a
   hardcoded or stale port.
2. **Startup ordering:** The measurement session may attempt to connect to
   signal-gen before it's ready, without adequate retry logic.
3. **Mode-dependent availability:** Signal-gen may only become reachable
   after GM switches to measurement mode, but the session tries to verify
   connectivity before the mode switch completes.

### Relationship to F-262

F-262 tracked filter verification failures (min-phase check). The cepstral
re-synthesis fix in D-262 resolved the filter verification issue. However,
the same tests also depend on the measurement session being able to start,
which requires signal-gen connectivity. The 503 failure was previously
masked by the F-262 xfails.

This is a separate defect from F-262 because the failure mode is different:
signal-gen connectivity at session startup, not filter quality at
verification time.

### Impact

E2E measurement session tests are xfailed. The measurement pipeline works
in unit/integration tests (T1 passes) but cannot complete a full E2E
session in the local-demo stack.
