## F-212: 55 JS 404 errors for missing static resources across all web UI tabs (OPEN)

**Filed:** 2026-03-29
**Severity:** Medium (no functional impact observed, but 404 noise pollutes browser console and server logs)
**Status:** OPEN
**Affects:** `src/web-ui/static/` — missing JS, CSS, or image resources referenced by HTML templates
**Found by:** QE (2026-03-29, exploratory testing with Playwright MCP — 55 errors across all tabs)
**Related story:** US-076 (Web UI visual polish), US-089-098 (stories in TEST)

### Description

During QE exploratory testing of the local-demo web UI (18/18 tab navigation
tests passing, 200+ tab switches), 55 HTTP 404 errors were logged for missing
static resources. The errors occur across all tabs, indicating missing JS
modules, CSS files, or image assets that are referenced in HTML templates but
do not exist in `src/web-ui/static/`.

### Impact

- No functional breakage observed (QE 18/18 pass)
- Browser console noise — 55 errors per session obscure real errors
- Server log pollution — uvicorn logs 404 responses for each missing resource
- May indicate dead references from removed features or incomplete asset
  bundling

### Suggested Investigation

1. Collect the full list of 404 URLs from QE's Playwright test output
2. Cross-reference against HTML templates to find the `<script>`, `<link>`,
   or `<img>` tags referencing non-existent files
3. Either create the missing files or remove the dead references

**Related:** US-076 (visual polish, completed), F-201 (prior local-demo UI
bug, verified fixed)
