## F-211: Reconciler pathological retry storm — 278 warnings in 2 seconds, no backoff (VERIFIED FIXED)

**Filed:** 2026-03-29
**Severity:** High (contributes to F-210 PW crash, excessive log noise, resource waste)
**Status:** VERIFIED FIXED (2026-03-29 — QE exploratory: 18/18 pass, 200+ tab switches, no crashes)
**Affects:** `src/graph-manager/src/reconcile.rs`, `src/graph-manager/src/registry.rs`
**Found by:** PM (log analysis of F-210 crash, 2026-03-29)
**Related story:** US-059 (GraphManager core), US-075 (local-demo stability)

### Description

During local-demo startup, the GM reconciler emits 278 "Required link
endpoint missing (will retry)" warnings in approximately 2 seconds
(lines 143-439 of `/tmp/local-demo.log`). The reconciler runs on every
PW registry node-add event and attempts to create all links in the routing
table. When link endpoints (ports) have not yet been enumerated by PW, the
reconciler logs a warning per missing endpoint and moves on.

The pathological behavior: each time a new node appears (Dummy-Driver,
Freewheel-Driver, USBStreamer, ada8200-in, convolver, convolver-out, UMIK-1,
loopback-sink, room-sim, room-sim-out, signal-gen = 11 events), the
reconciler checks all pending links. With 4 convolver-out -> USBStreamer
links pending and ports not yet visible, each node-add event produces 4
warnings. Across 11 nodes with additional port-add events, this compounds
to 278 warnings.

Additionally, at `16:20:44` (4 seconds after mode transition), WirePlumber
creates links that GM's reconciler immediately destroys as "stale" (not in
the routing table). WP recreates them. This fight produces 4 create/destroy
cycles in <1 second (lines 730-753).

### Impact

1. **Contributes to F-210** — the link churn and PW API call volume from
   278 reconciliation attempts likely contributes to PW buffer exhaustion
2. **Log noise** — 278 warnings in 2 seconds makes logs unreadable. On Pi,
   this would flood journald (related to F-093 which was the same class
   of bug, previously resolved)
3. **WP conflict** — GM and WP fighting over link ownership creates
   instability. The local-demo runs WP for node activation; GM manages
   links. But WP also tries to create default links, which GM destroys.

### Root Cause

1. **No backoff on retry** — the reconciler retries immediately on every
   registry event, even when the same endpoints were missing <100ms ago.
   It should suppress retries for recently-failed endpoints.
2. **No deduplication** — the same 4 missing-endpoint warnings repeat
   verbatim on every event. Deduplication or rate-limiting would reduce
   noise by 95%+.
3. **WP link management conflict** — WirePlumber's default linking policy
   creates links that conflict with GM's routing table. The local-demo
   should disable WP's default linking (e.g., via a `wireplumber.conf`
   that removes the `si-standard-link` module).

### Suggested Fix

1. **Reconciler backoff** — after detecting a missing endpoint, suppress
   retries for that specific endpoint for N seconds (e.g., 2s)
2. **Log deduplication** — downgrade repeated missing-endpoint warnings to
   DEBUG after the first occurrence per endpoint
3. **WP link policy** — in local-demo, configure WP to only activate nodes
   (not create links). This eliminates the GM-vs-WP link fight.
