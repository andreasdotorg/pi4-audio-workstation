## F-245: Measurement error UI shows raw Python/NumPy exception to user (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** High (user-facing raw exception, overlaps F-235)
**Status:** Open
**Affects:** US-097 room compensation web UI workflow, measurement tab
**Found by:** UX specialist screenshot review (session 7, UX-F3)

### Description

When measurement mode fails in local-demo, the measurement tab displays
the raw Python/NumPy exception:

> "zero-size array to reduction operation maximum which has no identity"

This is an internal implementation detail exposed directly to the user.
The error message is meaningless to the operator and provides no
actionable guidance.

### Context

This overlaps with F-235 (measurement mode broken in local-demo). The
root cause is likely the same — measurement mode fails because of missing
signal data or incorrect capture path configuration. However, even after
F-235 is fixed, the error handling should translate backend exceptions
into user-friendly messages.

### Impact

- Operator sees incomprehensible error during venue measurement
- No guidance on what went wrong or how to recover
- Undermines confidence in the measurement pipeline

### Fix

1. Catch NumPy/measurement exceptions in the measurement API endpoint
2. Translate to user-friendly message: "Measurement failed: no audio
   signal received from UMIK-1. Check that the microphone is connected
   and the correct input is selected."
3. Include actionable recovery steps in the error display

- F-235 (measurement mode broken in local-demo — root cause)
