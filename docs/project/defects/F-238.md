## F-238: Initial gains (-60/-64 dB) correct for Pi but questionable for simulated setup (OPEN)

**Filed:** 2026-03-31 (session 6)
**Severity:** Low
**Status:** Open
**Affects:** Local-demo gain defaults, signal levels in simulation
**Found by:** Owner (session 6, morning testing)

### Description

The local-demo convolver uses the production gain defaults: `Mult=0.001`
(-60 dB) for mains and `Mult=0.000631` (-64 dB) for subs. These values
are correct for the real Pi driving physical amplifiers and speakers, but
in the simulated local-demo environment they heavily attenuate the signal,
making meters and spectrum displays show very low levels.

### Context

US-075 audit bug #1 originally flagged gains at unity (1.0) as incorrect.
The owner directive was that production gains must match (0.001/0.000631).
This was fixed in commit `3e79abf`.

However, for simulation/development purposes, these extreme attenuation
values make it harder to visually verify audio flow — meters barely
register, spectrum is near the noise floor.

### Trade-off

- **Production-identical gains (current):** Faithful to production, but
  hard to see signal in local-demo meters/spectrum
- **Higher simulation gains:** Easier to verify audio flow visually, but
  diverges from production (violates US-075 mock boundary)

### Recommendation

This is a known trade-off, not a bug. The owner's mock boundary directive
is clear: production gains must match. The E2E tests can adjust gains via
the UI for verification purposes (per US-075 audit bug #1 owner decision).
No code change needed — document the expected behavior.
