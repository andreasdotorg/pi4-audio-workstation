## F-237: Speaker config activation behavior unclear / no venue config management (OPEN)

**Filed:** 2026-03-31 (session 6)
**Severity:** Medium (UX — operator confusion)
**Status:** Open
**Affects:** Web UI speaker config management, venue workflow
**Found by:** Owner (session 6, morning testing)

### Description

Activating a speaker configuration in the web UI is confusing. There is no
unified "venue config" concept that ties together speaker profiles, filter
coefficients, and gain staging. The operator cannot easily understand what
"activating" a speaker config actually does or how it relates to the
currently loaded filters and gains.

### Context

D-062 and D-063 define the target architecture: two-layer config model
(speaker profile + venue config), symlink-based activation, universal
audio gate. US-113 captures the implementation story (currently deferred).

The current UI (US-089) provides speaker config management but lacks the
venue config layer and the clear activation workflow described in D-062.

### Impact

Operator confusion during venue setup. Without a clear activation flow,
the operator may not understand:
- Which speaker config is currently active
- What happens when they click "Activate"
- How filters, gains, and corrections relate to the active config
- How to switch between venues or speaker setups
