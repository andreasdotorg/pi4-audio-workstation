## F-243: Negative phon values accepted client-side for ISO 226 (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Low
**Status:** Open
**Affects:** US-094 ISO 226 equal loudness compensation, client-side validation
**Found by:** QE exploratory Playwright session (session 7)

### Description

The custom phon input field in the ISO 226 equal loudness compensation
UI accepts negative values (e.g., -10 phon). ISO 226:2003 is defined for
the range 0-90 phon. The backend would reject these values at filter
generation time, but client-side validation is missing.

### Impact

Minor — backend catches the invalid value. Client-side validation would
provide immediate feedback and prevent unnecessary API calls.

### Fix

Add client-side input validation: min=0, max=90 (or max=100 to allow some
headroom). HTML `<input type="number" min="0" max="90">` or equivalent
JS validation.
