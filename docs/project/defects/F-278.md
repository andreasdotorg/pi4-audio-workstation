## F-278: test_audio_flow.py mislabeled as E2E -- all 8 tests bypass web UI

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** OPEN
**Affects:** `src/web-ui/tests/e2e/test_audio_flow.py`
**Origin:** QE E2E audit (session 14)

### Description

All 8 tests in `test_audio_flow.py` are located in the `e2e/` directory but
bypass the web UI entirely. They interact directly with the local-demo stack
via API calls or subprocess commands, not through a browser.

These are service-integration tests, not E2E tests. Their placement in the
`e2e/` directory inflates E2E coverage metrics and masks the absence of true
browser-driven E2E tests for audio flow functionality.

### Fix

Reclassify: move to `tests/service-integration/` (or equivalent non-E2E
directory). Optionally rewrite as browser-driven tests if the functionality
warrants true E2E coverage.
