## F-279: test_config_real.py mislabeled as E2E -- all 9 tests API-only

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** OPEN
**Affects:** `src/web-ui/tests/e2e/test_config_real.py`
**Origin:** QE E2E audit (session 14)

### Description

All 9 tests in `test_config_real.py` are located in the `e2e/` directory but
use direct API calls only -- no browser interaction. They validate config
endpoints against the local-demo stack but never exercise the web UI.

These are service-integration tests, not E2E tests.

### Fix

Reclassify: move to `tests/service-integration/`. Optionally rewrite with
browser interaction if config tab E2E coverage is needed (see US-145).
