## F-241: DRIVER PROTECTION shows "No speaker profile active" after activation (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Medium
**Status:** Open
**Affects:** US-092 per-driver thermal/mechanical protection display
**Found by:** QE exploratory Playwright session (session 7)

### Description

After successfully activating a speaker profile (bose-home), the DRIVER
PROTECTION section in the web UI still shows "No speaker profile active".
The profile activation itself succeeds (confirmed by "Activated: Bose Home
System" message and MUTE button toggle), but the protection display does
not update.

### Likely Cause

Stale state — the protection display component is not receiving the
profile activation event. Likely a missing WebSocket push notification
or the protection component not subscribing to the correct state update
channel. The activation and the protection display may use different
state sources.

### Impact

Operator confusion — they see a successful activation but the protection
panel says nothing is active. This could lead to uncertainty about whether
protection is actually engaged.

Note: THM status showing "--" is expected in local-demo mock mode (no
pw-cli actuation). This defect is specifically about the "No speaker
profile active" text not updating after activation.
