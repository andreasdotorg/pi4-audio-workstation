## F-262: E2E tests fail on CI — silence on all channels, GM reconciler not linking

**Filed:** 2026-04-04
**Severity:** Medium
**Status:** OPEN
**Affects:** CI T2 (E2E tests with local-demo PipeWire stack)
**Related:** CI run #335 (main at `fc40435`, PW 1.4.9)
**Origin:** CI test failure (owner directive: all CI failures tracked as defects)

### Description

E2E tests fail on CI with multiple symptoms:

1. **level-bridge reports all channels at silence (-120 dBFS):** No audio
   is flowing through the pipeline. All metering channels show floor level.
2. **Filter verification fails:** The filter-chain convolver is not
   receiving or processing audio.
3. **Measurement state stuck at 'error':** The measurement session cannot
   proceed because no signal is detected.

This is NOT a flaky test — QE confirms the failure is consistent and
needs investigation.

### Root Cause (suspected)

The GraphManager reconciler is not establishing the link topology in the
local-demo stack on CI. Without GM-managed PipeWire links connecting
signal sources to the filter-chain and output ports, audio cannot flow
through the pipeline. The individual services start (PipeWire, GM,
signal-gen, pcm-bridge, level-bridge, web-ui) but the link topology
that connects them is never created.

Possible sub-causes:
1. **GM reconciler timing:** The reconciler may not trigger correctly
   in the CI environment (race condition with PipeWire node registration).
2. **Missing PipeWire nodes:** The local-demo stack may not create all
   expected nodes that GM needs to reconcile.
3. **GM configuration:** The local-demo GM config may reference node
   names or ports that don't exist in the CI PipeWire graph.

### Fix

1. Investigate GM logs from CI run #335 — check whether the reconciler
   ran and what links it attempted to create.
2. Check PipeWire node registry at the time GM reconciler fires — are
   all expected nodes present?
3. If timing issue: add a readiness gate in the local-demo startup
   script that waits for all expected PipeWire nodes before starting GM.
4. If config issue: fix local-demo GM config to match the CI PipeWire
   graph topology.
