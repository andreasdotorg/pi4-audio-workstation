## F-250: Watchdog test flaky teardown — CancelledError during TestClient shutdown

**Filed:** 2026-04-04
**Severity:** Low
**Status:** OPEN
**Affects:** `tests/unit/test_measurement_integration.py::TestWatchdogTimeout::test_watchdog_triggers_abort_on_stall`
**Related:** CI run 23978434071
**Origin:** CI test failure (owner directive: all CI failures tracked as defects)

### Description

`test_watchdog_triggers_abort_on_stall` fails with `CancelledError` during
Starlette TestClient teardown when the watchdog aborts the measurement session.
The test itself passes (watchdog correctly triggers abort on stall), but the
async teardown of the TestClient raises `CancelledError` as background tasks
from the aborted session are still running when the client shuts down.

### Root Cause (suspected)

The watchdog abort cancels the measurement session's asyncio tasks. When the
Starlette TestClient tears down, it encounters these already-cancelled tasks
and propagates the `CancelledError` as an unhandled exception. This is a test
infrastructure issue, not a product bug — the watchdog behavior is correct.

### Fix

Suppress `CancelledError` during TestClient teardown in this specific test,
or add explicit cleanup of the measurement session before TestClient shutdown.
Alternatively, restructure the test to use a separate event loop for the
watchdog timeout to avoid task cancellation bleeding into teardown.

### Flaky Test Policy

Per US-070 AC #6: flaky tests must be fixed or quarantined with
`@pytest.mark.skip(reason="F-250: CancelledError during teardown")`. Do not
add retry logic.
