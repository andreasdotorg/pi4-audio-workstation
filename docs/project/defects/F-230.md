## F-230: No quantum change on mode switch — DJ should be 1024 (RESOLVED)

**Filed:** 2026-03-30 (session 6)
**Severity:** Medium
**Status:** RESOLVED (session 6, commit `00ff2f9` — quantum change added to GM mode switch handler)
**Affects:** GraphManager mode transitions, PipeWire quantum setting
**Found by:** QE (session 6, exploratory testing of local-demo)

### Description

When switching to DJ mode, the PipeWire quantum does not change to 1024 as
required by the production configuration. DJ mode needs quantum 1024 for
efficient convolution and CPU headroom for Mixxx. Live mode needs quantum
256 for low latency.

The quantum should be set via `pw-metadata -n settings 0 clock.force-quantum
1024` (DJ) or `256` (Live) as part of the mode transition.

### Impact

Functional — DJ mode may run at the wrong quantum (too low = unnecessary CPU
load, potential xruns; too high = unnecessary latency for live mode). This
was US-075 audit bug #4 (quantum settings) which was marked fixed, but QE
found the mode-switch path still doesn't change quantum dynamically.

### Context

US-075 bug #4 fixed the local-demo startup quantum to match production
defaults, but the GM mode-switch handler may not be updating quantum when
transitioning between modes.
