## F-216: Orphaned PipeWire/WirePlumber processes and zombie accumulation from local-demo/test runs (2026-03-29)

**Severity:** Medium
**Status:** Open
**Found in:** `scripts/local-demo.sh`, test harness scripts, Nix sandbox teardown
**Affects:** Development workflow, builder machine hygiene
**Found by:** Owner observation

### Description

The local-demo (`nix run .#local-demo`) and test scripts (`nix run .#test-*`)
don't reliably clean up child processes when killed (SIGINT/SIGTERM) or when
the Nix sandbox exits. This causes PipeWire, WirePlumber, graph-manager,
signal-gen, pcm-bridge, and related processes to be orphaned.

Over multiple development sessions, hundreds of zombie processes accumulate.
These are only clearable by reboot or manual `kill` sweeps. The zombies
consume PID space and confuse process inspection (`ps`, `htop`), and orphaned
PipeWire instances may hold audio device locks or socket files that interfere
with subsequent test runs.

### Root Cause

Multiple contributing factors:

1. **Shell trap handlers incomplete:** `local-demo.sh` has trap-based cleanup
   (F-203 fix added zombie reaping in `0c38e59`) but the cleanup doesn't
   cover all exit paths, particularly when the parent shell is killed by
   SIGKILL (e.g., Nix sandbox timeout) which prevents trap execution.
2. **Nix sandbox exit:** When `nix run` completes or is interrupted, the Nix
   sandbox may terminate the wrapper process but leave child processes that
   have daemonized or changed process groups.
3. **Process group management:** Child processes are not consistently placed
   in a process group that can be killed as a unit. Some processes (PipeWire,
   WirePlumber) may fork into their own sessions.
4. **Test scripts:** Individual `nix run .#test-*` targets may start PipeWire
   instances that outlive the test run.

### Expected

All child processes started by local-demo and test scripts are reliably
terminated when the parent script exits, regardless of exit method (normal
exit, SIGINT, SIGTERM, SIGKILL of parent, Nix sandbox teardown).

### Actual

~70+ zombie processes per session (F-203 count), potentially more over
extended development. Requires reboot to fully clear.

### Note

F-203 (committed `0c38e59`) addressed zombie cleanup within `local-demo.sh`
itself, but the broader issue of orphaned processes across all test entry
points and Nix sandbox boundaries remains.

### Possible Fixes

1. Use `setsid` + process group kill (`kill -- -$$`) for all child processes
2. PID file or cgroup-based tracking for reliable cleanup
3. A wrapper that uses `prctl(PR_SET_PDEATHSIG)` equivalent to ensure
   children die when the parent dies
4. The architect's US-111 local-demo redesign (eliminating WirePlumber,
   simplifying the process tree) will reduce the surface area

### Verified

Not yet.
