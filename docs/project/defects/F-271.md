## F-271: Generate D-031 protection filter WAVs for foh-passthrough.yml

**Filed:** 2026-04-06
**Severity:** High (safety — driver damage risk)
**Status:** OPEN
**Affects:** `foh-passthrough.yml` venue config — channels 1-4 (mains + subs)
**Related:** D-031 (driver protection filters), US-113 (first-boot active config), D-063 (universal audio gate)
**Origin:** AE blocking recommendation during US-113 review, revised to allow `dirac.wav` with tracked defect

### Description

The `foh-passthrough.yml` venue config uses `dirac.wav` (unity passthrough) on
channels 1-4 (mains + subs). This bypasses D-031 driver protection:

- **Mains (ch 1-2):** No highpass filter — subsonic content reaches wideband
  speakers unattenuated, risking cone excursion damage
- **Subs (ch 3-4):** No lowpass filter — high-frequency content reaches
  subwoofers unattenuated

Before this config is used in a real PA performance, protection filter WAVs
must be generated and channels 1-4 updated to reference them:

- `protection_hp.wav` — highpass protection for mains (per D-031 spec)
- `protection_lp.wav` — lowpass protection for subs (per D-031 spec)

### Interim Mitigation

- D-063 universal audio gate stays closed on first boot (`Mult=0.0` on all
  channels) — operator must explicitly open the gate
- Safety comment in `foh-passthrough.yml` warns against use without protection
  filters
- Operator awareness: FOH passthrough is for venue speaker systems where the
  venue's own processing chain provides driver protection

### Resolution

Generate protection filter WAVs using `crossover.py` and place them in
`configs/coeffs/`. Update `foh-passthrough.yml` channels 1-4 to reference
protection filters instead of `dirac.wav`.

### Blocked By

Nothing — can be implemented anytime using `crossover.py` and placed in
`configs/coeffs/` per Architect guidance.

### Verification

- Protection filter WAVs exist in `configs/coeffs/`
- `foh-passthrough.yml` references protection filters on channels 1-4
- AE confirms filter parameters match D-031 spec
- Audio passes through protection filters without audible coloration in the
  passband (flat response above/below crossover point)
