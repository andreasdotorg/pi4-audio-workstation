## F-282: test_mute_e2e.py partially mislabeled -- 5/8 tests API-only

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** OPEN
**Affects:** `src/web-ui/tests/e2e/test_mute_e2e.py`
**Origin:** QE E2E audit (session 14)

### Description

5 of 8 tests in `test_mute_e2e.py` use direct API calls only -- no browser
interaction. 3 tests have partial browser interaction but do not fully exercise
the mute UI flow through the browser.

### Fix

Reclassify the 5 API-only tests: move to `tests/service-integration/`.
Rewrite the 3 partial tests to use full browser interaction (click mute button,
verify VU meters go silent/recover). See US-144 for the browser-driven mute
E2E coverage story.
