## F-270: Transfer Function tab broken in local-demo (RESOLVED)

**Severity:** Medium (feature non-functional in dev/test environment)
**Status:** Resolved/Fixed (PR #21)
**Found in:** Local-demo stack (PI_AUDIO_MOCK=0), session 14
**Affects:** US-120 (Transfer Function feature), local-demo testing
**Found by:** Worker during E2E test development

**Description:** The Transfer Function (TF) tab in the web UI showed no data
when running against the real local-demo stack (PI_AUDIO_MOCK=0). The TF
WebSocket handler (`/ws/transfer-function`) requires two PCM sources:
`monitor` (reference, convolver output) and `capture-usb` (measurement, UMIK-1
input). Local-demo only started one pcm-bridge instance (monitor on port 9090)
and did not set `PI4AUDIO_PCM_SOURCES` to include `capture-usb`. The handler
closed the WebSocket immediately with code 4004 ("source not found").

**Root cause:** `scripts/local-demo.sh` launched a single pcm-bridge in
monitor mode but never started a second instance in capture mode for the
UMIK-1 source. The `PI4AUDIO_PCM_SOURCES` environment variable was not set,
so the web UI defaulted to `{"monitor": "tcp:127.0.0.1:9090"}` only — no
`capture-usb` entry. The TF handler's real-mode code path was never exercised
because all US-120 tests ran in mock mode (PI_AUDIO_MOCK=1).

**Impact:**
- TF tab completely non-functional in local-demo (the primary dev/test
  environment for real-mode code)
- No test coverage caught this because US-120 tests all used mock mode
- The feature appeared to work only because mock mode bypassed the real
  PCM reader code path entirely

**Fix (3 parts):**
1. **local-demo.sh:** Added a second pcm-bridge instance in capture mode
   reading from `alsa_input.usb-miniDSP_Umik-1` (the room-sim UMIK-1
   source node, mono). Port uses US-131 offset scheme
   (`PORT_PCM_CAPTURE`). Set `PI4AUDIO_PCM_SOURCES` with both entries
   using computed port variables. Added port to manifest, cleanup
   patterns, and `REQUIRED_PORTS`.
2. **Backend integration tests:** New `test_tf_backend_integration.py`
   with 9 Python-level WebSocket tests exercising the TF handler against
   the real local-demo stack (no browser automation).
3. **Browser E2E tests:** New `test_transfer_function.py` with Playwright
   tests verifying user-observable outcomes: WS connects, status shows
   "streaming", blocks counter increments, magnitude canvas has content.

**Related:** US-120 (Transfer Function feature), L-054 (mock policy),
US-131 (parallel local-demo instances).
