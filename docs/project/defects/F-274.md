## F-274: Channel gain UI not updated after audio gate open

**Filed:** 2026-04-06
**Severity:** Medium
**Status:** OPEN
**Affects:** Web UI config tab -- channel gain display
**Found by:** Owner (Gabriela Bogk, 2026-04-06)
**Related:** US-113 (audio gate), D-063 (universal audio gate)

### Description

When the audio gate is opened (unmuted), the channel gain UI element on the
config tab does not update to reflect the new gain state. The gain value
changes in PipeWire (audio flows correctly after gate open), but the UI
continues to show the stale pre-gate-open value.

### Expected Behavior

After the audio gate is opened, the config tab's channel gain display should
update to reflect the current gain values (the post-ramp-up Mult values).

### Actual Behavior

The gain UI element remains at the old value (likely Mult=0.0 / muted state)
after the gate is opened. The audio is flowing but the UI does not reflect
the actual gain state.

### Likely Cause

The gate open operation (cosine ramp-up) changes Mult values via `pw-cli` at
runtime. The config tab UI may not be subscribed to gain change events, or the
WebSocket relay may not push gain updates after the ramp-up completes.

### Verification

1. Open the web UI config tab
2. Open the audio gate (unmute)
3. Channel gain UI elements update to reflect current Mult values
