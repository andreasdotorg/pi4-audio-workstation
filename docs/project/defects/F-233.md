## F-233: FilterChainCollector poll loop silently stops updating — push event desync (OPEN)

**Filed:** 2026-03-30 (session 6)
**Severity:** High (monitoring data goes stale without warning)
**Status:** Open
**Affects:** Web UI dashboard, filter-chain status, convolver state display
**Found by:** QE (session 6, exploratory testing of local-demo)

### Description

The `FilterChainCollector` poll loop silently stops updating after some
period of operation. The collector initially retrieves filter-chain state
(convolver nodes, gain values, etc.) but at some point stops refreshing.
The web UI continues to display the last-known state without any indication
that the data is stale.

This appears to be a push event desync — the collector may be relying on
PipeWire events that stop arriving, or an internal state machine gets stuck.

### Impact

Dashboard shows stale filter-chain data. Operator may believe gains are at
one value when they've actually changed, or that the convolver is active
when it has been reloaded. Safety-adjacent: stale gain display could mask
an unsafe gain level.
