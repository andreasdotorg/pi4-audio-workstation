## F-259: Measurement Python tests not in CI — no flake app or check target

**Filed:** 2026-04-04
**Severity:** Low
**Status:** OPEN
**Affects:** `src/measurement/tests/test_signal_gen_client.py`
**Related:** CI pipeline, measurement subsystem
**Origin:** CI coverage audit (owner directive: all CI gaps tracked)

### Description

The measurement Python tests (`src/measurement/tests/test_signal_gen_client.py`)
have no flake app target, no flake check target, and are not included in
CI at all. This is a single test file with limited scope, but it represents
untested code in CI.

### Root Cause

No `nix run .#test-measurement` (or similar) app was defined in `flake.nix`
for the measurement test suite. Without a flake target, CI cannot run these
tests.

### Fix

1. Add a flake app target for measurement tests (e.g.,
   `nix run .#test-measurement`).
2. Add the new target to CI T1 pipeline.
3. Alternatively, if the test file is small enough, fold it into
   test-all or another existing test suite.
