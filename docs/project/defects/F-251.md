## F-251: Visual regression snapshot mismatches on CI runners

**Filed:** 2026-04-04
**Severity:** High (upgraded from Medium per AD review — entire visual regression gate non-functional in CI)
**Status:** OPEN
**Affects:** Browser integration tests (`test-integration-browser` CI job)
**Related:** CI run 23978434071
**Origin:** CI test failure (owner directive: all CI failures tracked as defects)

### Description

4 visual regression tests fail due to screenshot mismatches between locally
generated reference images and CI runner output:

| Test | Screenshot | Mismatch % |
|------|-----------|------------|
| dashboard view | `dashboard-view.png` | 43% |
| system view | `system-view.png` | 44% |
| measure stub | `measure-stub.png` | 14% |
| midi stub | `midi-stub.png` | 10% |

### Root Cause

Reference screenshots were generated on the developer's local machine (likely
different font rendering, DPI, Chromium version, or GPU compositing settings).
GitHub-hosted `ubuntu-latest` runners have different system fonts, no GPU
acceleration (software rendering), and potentially different Chromium/Playwright
versions. These environment differences produce pixel-level divergence in
rendered pages.

### Fix

Regenerate reference screenshots on the CI environment itself (GitHub-hosted
`ubuntu-latest` with the same Playwright/Chromium versions used in CI). Options:

1. **CI-generated baselines:** Add a workflow step that generates reference
   screenshots on first run, then compares on subsequent runs. Reference
   images committed from CI output, not local machine.
2. **Increase tolerance:** If pixel-perfect comparison is not required, add a
   percentage threshold (e.g., 5% mismatch allowed) to account for rendering
   differences.
3. **Structural comparison:** Replace pixel comparison with DOM-based or
   layout-based assertions that are rendering-engine-independent.

Option 1 is recommended — regenerate baselines from CI and commit them.
