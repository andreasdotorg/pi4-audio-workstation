## F-224: Mode switch failure in dashboard gives no user-visible error feedback (OPEN)

**Filed:** 2026-03-30 (session 6)
**Severity:** Low
**Status:** Open
**Affects:** Dashboard status bar mode switcher
**Found by:** worker-1 (during US-075 Bug #3 Mixxx substitute testing)

### Description

When switching modes via the dashboard status bar dropdown (e.g., standby
to DJ), if the switch fails (GM returns an error), the only user feedback is
a brief red outline on the mode badge that disappears after 1.5 seconds. The
actual error message from the backend is caught but never displayed.

The backend returns proper error details (HTTP 502, `gm_mode_switch_failed`
with a `detail` string explaining what went wrong, e.g., "required link
endpoint missing"). The frontend `.catch` handler discards this information.

### Reproduction

1. Start local-demo without a Mixxx node (or any mode that has unresolvable
   link endpoints)
2. Click the mode badge in the status bar
3. Select "DJ" from the dropdown
4. Observe: red outline flashes on badge for 1.5s, then disappears
5. No error message, no toast, no indication of what went wrong

### Location

Frontend: `src/web-ui/static/js/statusbar.js` lines 537-542:

```javascript
.catch(function (err) {
    var badge = document.getElementById("sb-mode");
    if (badge) {
        badge.style.outline = "2px solid red";
        setTimeout(function () { badge.style.outline = ""; }, 1500);
    }
})
```

Backend (working correctly): `src/web-ui/app/test_tool/routes.py` lines
469-473 returns `{"error": "gm_mode_switch_failed", "detail": "<reason>"}`.

### Expected Behavior

The UI should display the error reason to the user, e.g., via a toast
notification or inline error message below the mode dropdown. The error
detail from the backend response should be included so the user understands
why the switch failed.

### Fix Suggestion

Parse the error response JSON in the `.catch` handler and display a toast
or status message with the `detail` field. The red outline can remain as a
visual indicator, but the error text must also be shown.
