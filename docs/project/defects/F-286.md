# F-286: Post-merge regressions — duplicate UMIK + broken TF pane

**Status:** RESOLVED
**Severity:** High
**Reported:** 2026-04-06
**Resolved:** 2026-04-06 (commit bd3d05c0)
**PR:** #29

## Symptoms

After merging all open PRs (D-262, F-270, and others) to main:

1. **Duplicate UMIK-1 in test source dropdown:** Two UMIK entries appeared
   — "UMIK-1 (Live)" (virtual `umik1` from D-262) and "UMIK-1 (USB capture)"
   (real `capture-usb` from F-270).

2. **TF (Transfer Function) pane broken on Pi:** The TF WebSocket immediately
   closed with code 4004 ("Unknown meas PCM source: 'capture-usb'"). The TF
   tab appeared non-functional.

## Root Cause

Both regressions stem from the interaction between D-262 and F-270 after merge:

- **D-262** added a virtual `umik1` source that extracts channel 3 from the
  monitor PCM stream, plus a `capture-usb` label in `getSourceLabel()`.
- **F-270** added a real `capture-usb` pcm-bridge instance (reading from the
  UMIK-1 ALSA node) and set up `PI4AUDIO_PCM_SOURCES` in `local-demo.sh`.

**Symptom 1:** `populateSourceOptions()` unconditionally added the virtual
`umik1` option whenever `monitor` was present, without checking whether the
real `capture-usb` source already existed. Both entries appeared in the
dropdown after merge.

**Symptom 2:** The NixOS `web-ui.nix` service was never updated to set the
`PI4AUDIO_PCM_SOURCES` environment variable. Only `local-demo.sh` configured
it. Without it, the TF WebSocket backend fell back to a hardcoded default
containing only `{"monitor": "tcp:127.0.0.1:9090"}`. The TF handler requires
both `monitor` (reference) and `capture-usb` (measurement) sources, so it
closed with 4004 when `capture-usb` was missing.

## Fix

1. **`src/web-ui/static/js/test.js`:**
   - `populateSourceOptions()`: Only add virtual `umik1` when real
     `capture-usb` is absent (`sources.indexOf("capture-usb") < 0`).
   - `applyMeasurementDefaults()`: Prefer `capture-usb` over `umik1`
     when available for measurement mode auto-selection.

2. **`nix/nixos/services/web-ui.nix`:**
   - Auto-generate `PI4AUDIO_PCM_SOURCES` JSON from the enabled
     `pcm-bridge.instances` NixOS config (maps instance name to
     `tcp:127.0.0.1:<port>`).
   - Set `PI4AUDIO_PCM_CHANNELS=8`.
