## F-258: level-bridge Rust tests missing from CI and worker pre-push

**Filed:** 2026-04-04
**Severity:** Medium
**Status:** OPEN
**Affects:** CI T1 pipeline (`.github/workflows/ci.yml`), worker pre-push commands
**Related:** level-bridge service (production, always-on), `nix run .#test-level-bridge`
**Origin:** CI coverage audit (owner directive: all CI gaps tracked)

### Description

The level-bridge Rust crate has 57 tests and a working flake app target
(`nix run .#test-level-bridge`), but this target is not called anywhere:

1. **CI T1 job** runs test-all, test-pcm-bridge, test-signal-gen, and
   test-integration-browser — but NOT test-level-bridge.
2. **Worker pre-push commands** (in worker.md or development HOWTO) do
   not include test-level-bridge in the test list.

This is a real coverage gap: level-bridge is a production service
(3 instances, ports 9100-9102) providing 24-channel level metering.
Its tests are silently skipped in both CI and manual workflows.

### Root Cause

The test target exists but was not included when the CI pipeline (US-070)
and worker test commands were set up. Likely an oversight during the
transition from the original 2-job CI to the 4-tier restructure.

### Fix

1. Add `nix run .#test-level-bridge` as a step in CI T1 (after
   test-signal-gen, before test-integration-browser).
2. Add test-level-bridge to the worker pre-push test list in
   development documentation.
