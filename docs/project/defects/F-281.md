## F-281: test_filter_deploy_e2e.py mislabeled as E2E -- all 9 tests API-only

**Filed:** 2026-04-06
**Severity:** Medium (test infrastructure)
**Status:** OPEN
**Affects:** `src/web-ui/tests/e2e/test_filter_deploy_e2e.py`
**Origin:** QE E2E audit (session 14)

### Description

All 9 tests in `test_filter_deploy_e2e.py` are located in the `e2e/` directory
but use direct API calls only -- no browser interaction. They validate filter
deployment endpoints against the local-demo stack but never exercise the web UI.

These are service-integration tests, not E2E tests.

### Fix

Reclassify: move to `tests/service-integration/`.
