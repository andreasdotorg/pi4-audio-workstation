## F-223: Auth middleware intercepts all requests in local-demo despite PI4AUDIO_AUTH_DISABLED=1 (OPEN)

**Filed:** 2026-03-30 (session 5)
**Severity:** High (blocks owner acceptance of 7 stories)
**Status:** Open
**Affects:** US-090, US-092, US-093, US-094, US-095, US-096, US-097 (all REVIEW stories rejected)
**Found by:** Owner (session 5)

### Description

The local demo (`local-demo.sh`) sets `PI4AUDIO_AUTH_DISABLED=1` at line 446,
but the US-110 auth middleware intercepts all HTTP requests regardless. Hitting
`/` redirects to `/auth/login?next=/` which returns 404 (auth routes not yet
registered in local-demo mode).

Owner quote: "exactly none of those stories have been end to end tested."

### Reproduction

1. Run `local-demo.sh` (or `nix run .#serve`)
2. Open browser to `http://localhost:8080/`
3. Observe redirect to `/auth/login?next=/` which 404s
4. No web UI content is accessible

### Root Cause

The auth middleware added by US-110 (T-110-04, commit `4efb3b8`) does not
check the `PI4AUDIO_AUTH_DISABLED` environment variable before intercepting
requests. The middleware unconditionally redirects unauthenticated requests
to the login page.

### Impact

All 7 REVIEW stories (US-090, US-092-097) rejected by owner because the
local demo is completely broken — no page can be accessed for E2E testing.

### Fix

The auth middleware must check `PI4AUDIO_AUTH_DISABLED=1` and skip
authentication when disabled. This is a US-110 implementation bug.
