## F-236: UMIK-1 spectrum in test tab shows frequency-dependent rolloff (RESOLVED)

**Filed:** 2026-03-31 (session 6)
**Severity:** Medium
**Status:** RESOLVED
**Resolved:** 2026-03-31 (session 7, commit `a06dd18`)
**Affects:** Web UI test tab spectrum display, UMIK-1 capture visualization
**Found by:** Owner (session 6, morning testing)

### Description

The UMIK-1 spectrum display in the test tab shows frequency-dependent
rolloff — higher frequencies are increasingly silent. This is similar to
a previously fixed bug in the main dashboard spectrum display.

### Root Cause

Stale 48-byte coefficient stubs caused the convolver to fail silently.
When the convolver config was extended from 4ch to 8ch (US-115), the
placeholder coefficient files were not regenerated at the correct length.
The convolver loaded these stubs and produced incorrect frequency response.

### Fix

Commit `a06dd18` replaced the stale 48-byte coefficient stubs with proper
16384-sample coefficients. The convolver now processes all channels correctly.
