## F-254: Driver YAML Qts consistency check failures

**Filed:** 2026-04-04
**Severity:** Low
**Status:** OPEN
**Affects:** `tests/unit/test_driver_validation.py::TestRealDriverRecords::test_sample_real_drivers_pass`
**Related:** CI T1 test suite
**Origin:** CI test failure (owner directive: all CI failures tracked as defects)

### Description

Two driver YAML configuration files fail the Qts consistency check
(deviation exceeds 5% threshold):

| Driver | Deviation |
|--------|-----------|
| morel-tsct1104 | 5.6% |
| prv-audio-6mr500-ndy-4 | 6.6% |

The Qts consistency check verifies that Qts computed from Qes and Qms
matches the declared Qts value in the driver YAML. These two drivers
exceed the 5% tolerance.

### Root Cause (suspected)

Either:
1. The 5% tolerance is too tight for real-world driver data (manufacturer
   specs have inherent measurement uncertainty)
2. The driver YAML data contains transcription errors in Qes, Qms, or Qts

### Fix

1. **Verify driver data:** Cross-check morel-tsct1104 and prv-audio-6mr500-ndy-4
   YAML files against manufacturer datasheets. Correct any transcription errors.
2. **Loosen tolerance:** If the data is correct, increase the Qts consistency
   threshold from 5% to 7% (accommodates manufacturer measurement uncertainty).
3. **Per-driver tolerance:** If only specific drivers have looser specs, add a
   per-driver tolerance override rather than loosening the global threshold.

Option 1 is recommended — verify data before loosening tolerances.
