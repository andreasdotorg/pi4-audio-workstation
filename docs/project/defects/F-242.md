## F-242: Negative sensitivity returns opaque "write_failed" error (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** Low
**Status:** Open
**Affects:** US-093 amplifier sensitivity calibration, validation UX
**Found by:** QE exploratory Playwright session (session 7)

### Description

Entering a negative sensitivity value (-1 Vrms) in the amplifier edit
form returns the opaque error "Error: write_failed" instead of a
field-specific validation message. By contrast, entering power=0 correctly
returns "must be a positive number" — a clear, field-specific message.

### Impact

Minor UX inconsistency. The validation catches the invalid value (backend
rejects it), but the error message does not help the operator understand
what is wrong or which field caused the error.

### Fix

Add field-specific validation for sensitivity (must be positive) matching
the pattern used for power validation.
