## F-244: All entity DELETE buttons in config tab lack confirmation dialogs (OPEN)

**Filed:** 2026-03-31 (session 7)
**Severity:** High (usability — single-click destructive actions during live show setup)
**Status:** Open
**Affects:** Config tab entities: speaker profiles (US-089), amplifiers (US-093), DACs (US-093), speaker identities (US-039). Cross-cutting UX pattern.
**Found by:** UX specialist screenshot review (session 7, UX-F9)

### Description

DELETE buttons across all entity types in the config tab execute
immediately on single click with no confirmation dialog. This affects:

- Speaker profiles (US-089)
- Amplifier configurations (US-093)
- DAC configurations (US-093)
- Speaker identity files (US-039)

During venue setup, an accidental click could delete any of these
entities with no undo path.

### Context

This violates standard usability practice for destructive actions. All
destructive operations should require explicit confirmation, especially
in a live/event context where the operator may be under time pressure
and distracted. The pattern is the same across all entity types — a
single fix template can be applied universally.

### Impact

- Accidental entity deletion during show setup could disrupt event
- No undo mechanism — deleted entities must be manually recreated
- Particularly dangerous on touch interfaces (VNC on tablet) where
  accidental taps are more likely
- Amplifier and DAC deletion could silently invalidate dependent
  speaker profiles and thermal calculations

### Fix

Add a confirmation dialog before ALL DELETE buttons in the config tab:
"Delete [entity type] '[name]'? This cannot be undone." with Cancel
(default) and Delete buttons. Apply the same pattern used for F-046
(quantum change confirmation).
